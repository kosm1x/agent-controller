## Three patterns observed in v7.6

### Pattern 1 — Round-2 audits catch contract-level drift introduced by round-1 fixes

Across Spines 1, 2, 4, and the SDK cutover, the same shape kept appearing: round-1 audit catches a Critical, round-1 fix addresses it but introduces a new contract-level inconsistency, round-2 audit catches that drift. Examples: Spine 1's negation guards came in classes (catch "no", "ni" follows in next round); Spine 4's bulk classification table had a 4.3% misclassification floor that only round-2 name-pattern sweep surfaced; SDK cutover's C1 fix exposed heavy/swarm `actualModel` drop only after round-2 traced the full plumbing.

**Lesson**: 2-round audit cadence is not optional for bundles unifying parsers, classification tables, or telemetry surfaces. Round-1 alone leaves drift one cycle removed. Codified in `feedback_round2_audit_pattern.md`.

### Pattern 2 — Producer/consumer coupling tests are the load-bearing observability invariant

Spine 2's `fullCount` finding (off-by-one between producer scope.ts and consumer router.ts:437) and Spine 5's M4 tool-name parity test both followed the same shape: an observability surface that _named_ a thing in one place and _read_ it in another, with no test pinning the equivalence. The fix in each case wasn't more counting — it was a single-source-of-truth helper plus a test that fails loudly on rename.

**Lesson**: every observability surface needs both ends pinned in one test. Adding a counter or label without the coupling test is shipping the silent-drift class from day 1.

### Pattern 3 — Independent spines + per-spine audit-in-bundle = compressed timeline

Original V7.6-GUIDE estimate: 4 weeks. Actual: 10 days. The compression came from two factors. First, all six work-spines were independently shippable (deliberate phase architecture) — three were closed same-day 2026-05-08 (Spines 1, 2, 4) and the other three (5, 6, plus Spine 1's tail) followed within a week. Second, the round-2 audit cadence catches contract drift in-bundle, so no rework cycles materialized — every closure was final.

**Lesson**: phase-architecture independence is more valuable than spine-size estimation accuracy. The slowest spine sets the calendar, not the sum. v7.7's 7-spine PREPLAN should aim for similar independence; the spec already orders dependencies (Spines 1-3 independent, 4-6 layer on top).
