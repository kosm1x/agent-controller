> **✅ COMPLETE 2026-07-12 (same session, via Phase 1b bisect).** The initial attempt gate-FAILED twice and was reverted; the bisect then found the truth: the "regression" was the lean-system-prompt rollout (0.3.153) removing an **operator-config leak** — with `settingSources` unset, every Jarvis call had carried `~/.claude/CLAUDE.md` + rules + MEMORY.md (~23k tokens), and the 57.95 incumbent was measuring that contamination. Full story: PROJECT-STATUS 2026-07-12b/c. Phases 2–3 SDK-dependent items are UNBLOCKED. Note for Phase 3.1: cache-prefix economics changed (~23k smaller stable prefix + 1h-TTL writes) — re-measure before choosing a fix option. Follow-up: sweep other SDK consumers on this VPS for the same unset-settingSources leak.

Everything strategic downstream (Fable 5 access, refusal handling, budget signals) requires this. Highest-touch item; dedicated session with `npm run eval:gate -- --run` before/after and the option to hold the old lockfile for instant revert.

**Breaking changes to absorb** (from the changelog recon):

- 0.2.113: SDK spawns a native Claude Code binary; `options.env` **replaces** `process.env` (we pass `CLAUDE_CODE_AUTO_COMPACT_WINDOW` + client-app via `env` — must now pass through everything the subprocess needs).
- 0.3.142: v2 session API removed; MCP servers connect non-blocking by default (`alwaysLoad: true` to opt out) — our single in-process `jarvis` MCP server must be connected before first tool call.
- 0.3.143: `@anthropic-ai/sdk` / MCP SDK become peerDeps — add explicitly.
- 0.3.162: Grep/Glob built-ins replaced by embedded find/grep in Bash — we run `tools: []`, likely unaffected, verify.

**New capabilities to adopt in the same pass** (each maps to a known failure class):

- `stop_reason: "refusal"` + `stop_details` → deterministic handling instead of "model said nothing" (never-silent-reply floor family).
- `command_lifecycle` terminal frames incl. **`budget_exhausted`** → surfaces the exact signal our cost softcap plumbing is currently blind to mid-run.
- `Query.reinitialize()` transport recovery → replaces some circuit-breaker/retry hand-rolling.
- Per-MCP-server `request_timeout_ms` → aligns with our LLM-client-timeout-vs-proxy-ceiling lesson.
- Structured outputs at the SDK level (validated JSON agent outputs) → candidate replacement for the forced-structured-output-via-MCP-tool pattern in `v8-2/*` + `audit/critic.ts` (evaluate, don't rip out — the current pattern works).
