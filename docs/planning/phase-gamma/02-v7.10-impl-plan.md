# v7.10 — Universal File Conversion (`file_convert`) Impl Plan

Session 85 · 2026-04-20 · Phase γ S2.

## §0 Scope

One new deferred tool `file_convert` that dispatches to a fixed, whitelisted set of apt-installed FLOSS binaries. Closes 5 real gaps Jarvis has: `.epub/.mobi` → text, office docs ↔ docx/pdf/text, HEIC/AVIF/JXL → JPEG/PNG, arbitrary doc↔doc via pandoc, video frames for vision.

Reference: `reference_convertx.md` (ConvertX AGPL-blocked; rebuild-native verdict).

**Explicit non-goal**: audio transcription. Separate project (whisper.cpp or Deepgram).

## §1 Target delivery

- +1 tool handler: `src/tools/builtin/file-convert.ts` (~200 LOC)
- +1 test file: `src/tools/builtin/file-convert.test.ts` (~200 LOC)
- +1 apt installation step (≈2GB disk, one-time)
- +0 npm deps (pure `node:child_process.execFile`)
- Integration: builtin.ts register, UTILITY_TOOLS scope group, fast-runner WRITE_TOOLS set
- Scope: extend `utility` regex with conversion vocabulary (EN + ES)

## §2 Dispatch table (fixed, not LLM-inferred)

| Input extension(s)                                       | Target formats                | Binary        | Invocation                                                   |
| -------------------------------------------------------- | ----------------------------- | ------------- | ------------------------------------------------------------ |
| `.epub .mobi .azw .azw3 .fb2`                            | `txt pdf epub docx html md`   | ebook-convert | `ebook-convert INPUT OUTPUT`                                 |
| `.odt .rtf .doc .docx .ppt .pptx .xls .xlsx .pages .key` | `pdf docx html txt md`        | libreoffice   | `libreoffice --headless --convert-to FMT --outdir DIR INPUT` |
| `.md .html .rst .tex .org .adoc`                         | `html md pdf docx epub txt`   | pandoc        | `pandoc INPUT -o OUTPUT`                                     |
| `.heic .heif .avif .jxl .tiff .bmp .webp`                | `jpeg png webp`               | magick        | `magick INPUT OUTPUT`                                        |
| `.mp4 .mov .avi .mkv .webm`                              | `jpeg png` (frame extraction) | ffmpeg        | `ffmpeg -ss TIMESTAMP -i INPUT -vframes 1 -q:v 2 OUTPUT`     |

Target-format enum (Zod-validated): `txt pdf epub docx html md jpeg png webp`.

## §3 Security

Primary threat: `shell_exec`-style abuse of the binary dispatch to read arbitrary files or write outside sandboxes.

1. **No shell** — use `execFile(binary, [args...])` with arg array. No string concatenation. No user-controlled flags.
2. **Absolute path required** for `input_path`. Reject relative, `..`, and URL schemes.
3. **Source whitelist** — input must resolve under one of:
   - `/tmp/`
   - `/workspace/`
   - `/root/claude/jarvis-kb/`
   - `/root/claude/projects/`
   - `/root/claude/mission-control/public/docs/`
     Not `/root/claude/mission-control/src/` (prevent self-read of source via "convert" abuse path).
4. **Source must exist** and be a regular file (not symlink to outside whitelist, not directory).
5. **Target enum** — only the 9 entries above. Rejects anything else.
6. **Output path** — if `output_path` provided, must be under `/tmp/` or `/workspace/`. Default: `/tmp/file-convert-${randomUUID}.${target_format}`.
7. **Timeout** — 60s per call (libreoffice cold-start can hit 15s; 60s covers large PDFs).
8. **Output size cap** — read the produced file's size but don't read contents; return path + byte count.

## §4 Tool interface (ACI-friendly)

```ts
file_convert({
  input_path: string;        // Absolute path to source file
  target_format: "txt" | "pdf" | "epub" | "docx" | "html" | "md" | "jpeg" | "png" | "webp";
  output_path?: string;      // Optional override; defaults to /tmp/file-convert-{uuid}.{target}
  timestamp_sec?: number;    // Video frame extraction only; seconds into source. Default 1.
}) → string (human-readable summary with absolute output path, bytes, duration)
```

Description in tool definition teaches the dispatch:

> Convert a file between formats using standard FLOSS CLI tools (calibre, libreoffice, pandoc, imagemagick, ffmpeg). Use when you need to read a format Jarvis can't parse natively (.epub, .mobi, .odt, .rtf, .pages, HEIC/AVIF images, video frames) or produce a specific output format. The source extension determines which binary runs; target_format is enum-constrained. Output is written to /tmp by default.

## §5 Sequence (compact 16 steps)

1. Install apt binaries (calibre libreoffice pandoc imagemagick libvips-tools).
2. Verify versions (`pandoc --version`, `libreoffice --version`, `ebook-convert --version`, `magick --version`).
3. Implement `src/tools/builtin/file-convert.ts`.
4. Write tests `src/tools/builtin/file-convert.test.ts` (mock execFile, cover dispatch + security + error paths).
5. Register in `src/tools/sources/builtin.ts` (both arrays).
6. Extend `UTILITY_TOOLS` in `src/messaging/scope.ts` with `file_convert`.
7. Extend utility scope regex with conversion vocabulary (EN: convert/transform/extract/frame ES: convertir/convierte/extraer/transformar).
8. Add `file_convert` to `WRITE_TOOLS` set in `fast-runner.ts` (hallucination guard — tool writes a file).
9. Add scope-regex test cases (positive + negative).
10. `npm run typecheck` + `npm test` — all green.
11. `./scripts/deploy.sh`.
12. Live smoke: `.md→.html` via pandoc, `.png→.jpeg` via magick, video frame via ffmpeg.
13. QA audit round 1. Fix.
14. QA audit round 2. Fix.
15. Update PROJECT-STATUS, V7-ROADMAP, EVOLUTION-LOG, memory.
16. Commit + push + PR + merge.

## §6 Risks

| Risk                                                                     | Mitigation                                                                                                                                   |
| ------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------- |
| libreoffice cold start is slow (5-15s) — causes timeout on first call    | 60s timeout. Doc in tool description that first conversion may be slow.                                                                      |
| calibre installation pulls huge dep tree (~500MB)                        | Accept — disk has 28GB free. One-time install.                                                                                               |
| magick/ImageMagick `policy.xml` may block PDF/PS operations by default   | Not needed at MVP — we only convert raster images. Keep PDF conversion path via libreoffice/pandoc.                                          |
| ffmpeg frame extraction on large videos is slow                          | `-ss` flag before `-i` seeks instantly. 60s timeout suffices for any video.                                                                  |
| Binary not found at runtime (e.g., apt uninstalled)                      | Preflight `existsSync` via `which`-style check before invoking; return clear error "calibre not installed — run `apt install calibre`".      |
| `file_convert` is called with untrusted `input_path` from a user message | Whitelist enforces sandboxing. All sources are Jarvis-readable areas; all outputs are under `/tmp` or `/workspace`. No privilege escalation. |
| Pandoc target format mismatch (e.g. `.rst → jpeg` doesn't make sense)    | Dispatch table rejects unsupported input/target pairs. Error message lists valid targets for the detected source.                            |

## §7 Success criteria

1. Zero typecheck errors, all tests pass (≥2970).
2. Deploy succeeds; boot has no new errors.
3. Live smoke: 3 real conversions (text, image, video frame) produce valid output files.
4. QA audit round 2 clean PASS on the fixes.
5. Rollback plan: revert the commit — no DB migrations, no config that needs undoing. Tool just drops out.

## §8 Deferrals (v7.10.1+)

- **Audio transcription** (whisper.cpp or Deepgram API) — separate project entirely.
- **Batch conversion** — tool converts one file at a time at MVP. Multi-file call could be v7.10.1.
- **Custom pandoc templates / reference-doc** — use pandoc defaults. Power users can post-process.
- **Format auto-detection from magic bytes** — rely on extension; users pass correct extensions.
- **Streaming / chunked conversion** — irrelevant for our use case; file sizes are bounded by user uploads.
- **Hindsight logging** — file_convert writes to /tmp not DB; no persistent log needed beyond tool-call history in Hindsight.
