## Architecture

```
market_chart_render(symbol, interval, lookback_bars, indicators[], signals?[], output_path?)
  ↓
  DataLayer.getDaily/getWeekly(symbol, lookback_bars)
  ↓
  indicators.computeAll(bars, indicators[])
  ↓
  chartHtml(bars, indicatorSeries, signals) → string (inlines lightweight-charts.standalone.js + data + init script)
  ↓
  chart-render.renderToPNG(html, width, height) → Uint8Array
    ├─ playwright.chromium.launch({ headless: true })
    ├─ page.setContent(html, { waitUntil: "networkidle" })
    ├─ page.waitForSelector("#chart canvas") + small delay for drawing
    ├─ page.screenshot({ clip: chart-div-bbox })
    └─ browser.close() in finally
  ↓
  writeFileSync(output_path, png)
  ↓
  returns { path, width, height, bars_rendered }

market_chart_patterns(chart_png_path | {symbol, interval, lookback_bars})
  ↓
  if input is symbol, call market_chart_render internally
  ↓
  vision LLM call: infer({ messages: [..., image_content], ...modelOpts })
  ↓
  parse structured JSON { pattern, confidence, candle_range, rationale }
  ↓
  persistChartPattern(db, { symbol, interval, pattern_label, confidence, candle_start, candle_end, png_path })
  ↓
  returns { pattern, confidence, rationale, pattern_id, png_path }
```

## Schema

```sql
CREATE TABLE IF NOT EXISTS chart_patterns (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  symbol        TEXT NOT NULL,
  interval      TEXT NOT NULL CHECK (interval IN ('daily','weekly')),
  pattern_label TEXT NOT NULL,
  confidence    REAL NOT NULL CHECK (confidence >= 0 AND confidence <= 1),
  candle_start  INTEGER,  -- index into the rendered window (nullable for whole-chart patterns)
  candle_end    INTEGER,
  png_path      TEXT,
  rationale     TEXT,
  detected_at   INTEGER NOT NULL DEFAULT (unixepoch())
);
CREATE INDEX IF NOT EXISTS idx_chart_patterns_symbol_detected ON chart_patterns(symbol, detected_at DESC);
```

## Trend-channel algorithm (pure TS)

1. **Pivots**: detect local highs (bar[i].high > neighbors ±window) and local lows (bar[i].low < neighbors ±window). Default window=3.
2. **Channel fit**: least-squares regression on pivot highs (upper line) and pivot lows (lower line). Return `{slope, intercept, r2}` per line.
3. **Slope quantification**: `slope_per_bar = (lastY - firstY) / (n-1)`; classify as `uptrend | downtrend | sideways` by slope magnitude vs bar ATR.
4. **Consolidation zone**: channel width < `k * ATR` over last N bars → flag as consolidation. k default 2.0, N default 20.
5. Export `fitTrendChannel(bars): TrendChannel` and `detectConsolidation(bars, opts?): boolean`.

## Security / blast radius

- `symbol` passed into HTML template must be escape-encoded (`&quot;`, `&lt;`) — otherwise a malicious symbol could inject `<script>` into chart title. Use JSON.stringify for all data insertions into the template; the library ingests JSON natively.
- `output_path` whitelist: only under `/tmp/jarvis/charts/` or `data/charts/`.
- `chart_png_path` input (patterns tool): lstat + realpath validation, mirror v7.10 `file_convert` pattern — prevent symlink-escape to `/etc/shadow`.
- Playwright launch args: reuse `STEALTH_LAUNCH_ARGS` subset (sandbox disabled for root, `--disable-dev-shm-usage` to avoid `/dev/shm` exhaustion documented in pre-explore).
- Outer timeout 30s on chart-render; 60s on patterns tool (includes LLM call).
