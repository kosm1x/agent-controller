# v7.11 — Jarvis Teaching Module Impl Plan

Session 89 · 2026-04-21 · Phase γ S6.

## §0 Scope

Enable "teach me X" end-to-end: decompose a topic into ordered units, drive a scoped chat per unit, quiz adaptively, run a Socratic explain-back, score mastery with persisted evidence, and schedule spaced-repetition reviews. Morning ritual surfaces what's due today; EOD ritual applies mastery deltas.

Reference: `reference_deeptutor.md`. DeepTutor supplies ~30% (plumbing prompts + state machine). The 70% load-bearing work — learner model, SM-2 spaced repetition, misconception detection, Socratic grading, prerequisite graph, adaptive difficulty — is Jarvis-native.

**Non-goals**: interactive HTML unit rendering, generated videos (v7.4), multi-user progress, cross-project collab.

## §2 Data model

### 2.1 learning_plans

```sql
CREATE TABLE IF NOT EXISTS learning_plans (
  plan_id      TEXT PRIMARY KEY,
  topic        TEXT NOT NULL,
  created_at   INTEGER NOT NULL DEFAULT (unixepoch()),
  updated_at   INTEGER NOT NULL DEFAULT (unixepoch()),
  status       TEXT NOT NULL CHECK(status IN ('active','paused','completed','archived'))
                              DEFAULT 'active',
  current_unit INTEGER NOT NULL DEFAULT 0,
  notes        TEXT
);
CREATE INDEX IF NOT EXISTS idx_learning_plans_status ON learning_plans(status, updated_at DESC);
```

### 2.2 learning_plan_units

```sql
CREATE TABLE IF NOT EXISTS learning_plan_units (
  plan_id                TEXT NOT NULL REFERENCES learning_plans(plan_id) ON DELETE CASCADE,
  unit_index             INTEGER NOT NULL,
  title                  TEXT NOT NULL,
  summary                TEXT NOT NULL,
  predicted_difficulties TEXT,  -- JSON array of strings
  prerequisites          TEXT,  -- JSON array of unit_indexes (local) OR concept keys (external)
  status                 TEXT NOT NULL CHECK(status IN ('locked','ready','in_progress','mastered','skipped'))
                                      DEFAULT 'locked',
  mastery_score          REAL NOT NULL DEFAULT 0 CHECK(mastery_score >= 0 AND mastery_score <= 1),
  PRIMARY KEY (plan_id, unit_index)
);
CREATE INDEX IF NOT EXISTS idx_learning_plan_units_status
  ON learning_plan_units(plan_id, status, unit_index);
```

### 2.3 learner_model

```sql
CREATE TABLE IF NOT EXISTS learner_model (
  concept          TEXT PRIMARY KEY,
  first_seen       INTEGER NOT NULL DEFAULT (unixepoch()),
  last_seen        INTEGER NOT NULL DEFAULT (unixepoch()),
  confidence       REAL NOT NULL DEFAULT 0 CHECK(confidence >= 0 AND confidence <= 1),
  mastery_score    REAL NOT NULL DEFAULT 0 CHECK(mastery_score >= 0 AND mastery_score <= 1),
  evidence_quotes  TEXT,  -- JSON array {quote, session_id, ts}
  -- SM-2 state:
  ef               REAL NOT NULL DEFAULT 2.5 CHECK(ef >= 1.3),  -- easiness factor
  interval_days    INTEGER NOT NULL DEFAULT 0,
  repetitions      INTEGER NOT NULL DEFAULT 0,
  review_due_date  INTEGER  -- unix seconds; NULL = not scheduled
);
CREATE INDEX IF NOT EXISTS idx_learner_model_due
  ON learner_model(review_due_date) WHERE review_due_date IS NOT NULL;
```

### 2.4 learning_sessions

```sql
CREATE TABLE IF NOT EXISTS learning_sessions (
  session_id      TEXT PRIMARY KEY,
  plan_id         TEXT NOT NULL REFERENCES learning_plans(plan_id) ON DELETE CASCADE,
  unit_index      INTEGER NOT NULL,
  kind            TEXT NOT NULL CHECK(kind IN ('teach','quiz','explain_back','review')),
  started_at      INTEGER NOT NULL DEFAULT (unixepoch()),
  ended_at        INTEGER,
  mastery_delta   REAL,
  transcript_ref  TEXT,  -- optional pointer to messages/conversation id
  summary         TEXT
);
CREATE INDEX IF NOT EXISTS idx_learning_sessions_plan
  ON learning_sessions(plan_id, started_at DESC);
```

## §4 SM-2 algorithm (brief)

Quality ∈ {0..5} (0 = blank, 3 = pass, 5 = perfect).

- If `quality < 3`: `repetitions = 0; interval_days = 1`
- Else:
  - `repetitions += 1`
  - If `repetitions == 1`: `interval_days = 1`
  - Elif `repetitions == 2`: `interval_days = 6`
  - Else: `interval_days = round(interval_days * ef)`
- `ef = max(1.3, ef + 0.1 − (5 − quality) * (0.08 + (5 − quality) * 0.02))`
- `review_due_date = now + interval_days * 86400`

Mastery score derived (display-only): `min(1, (repetitions / 6) * (ef / 2.5))` — clamped [0,1].

## §5 The 6 tools (scope `teaching`, all `deferred: true`)

### 5.1 `learning_plan_create(topic: string, ceiling_units?: 2..8)`

LLM call with `designPrompt(topic)`. Expect JSON array of 2–5 units (bounded by ceiling). Parse → validate → create plan + insertUnits in a single transaction. Unit 0 status = `ready`, rest `locked`. Return `{plan_id, units: [{index, title, predicted_difficulties}]}`.

### 5.2 `learning_plan_advance(plan_id: string)`

Look up `current_unit`. If its `mastery_score >= 0.7`, mark it `mastered`, find next unit whose prerequisites are all mastered, set it `ready`, advance `current_unit`. If none, mark plan `completed`. Return `{next_unit: {index, title} | null, plan_status}`.

### 5.3 `learning_plan_quiz(plan_id: string, unit_index?: int, n?: 1..5)`

LLM call with `quizPrompt(unit, history)` where history = prior quiz questions for this unit (dedup). Default n=3. Adaptive difficulty: map current `mastery_score` to `target_difficulty ∈ {easy, medium, hard}` (`<0.3 → easy`, `0.3–0.7 → medium`, `>0.7 → hard`). Persist quiz session. Return questions + expected answers (hidden from user in display rendering, but kept for grading on next turn).

### 5.4 `learning_plan_explain_back(plan_id: string, unit_index?: int, user_explanation?: string)`

Two-mode:

- No `user_explanation` → emit Socratic prompt: "In your own words, how would you explain {unit.title} to someone who doesn't know it?" + record session started.
- With `user_explanation` → grade via LLM judge rubric (5 criteria: completeness, accuracy, structure, misconceptions flagged, confidence tone), produce quality 0..5. Update learner_model concepts. Close session. Return `{quality, mastery_delta, feedback}`.

### 5.5 `learning_plan_summarize(plan_id: string, unit_index?: int)`

LLM call with `summaryPrompt(transcript, unit)`. Parse `{concepts: [{name, evidence_quote, mastery_estimate}]}`. Upsert each concept into `learner_model` via SM-2 (quality derived from mastery_estimate: `round(mastery * 5)`). Update unit `mastery_score` to mean of concept masteries. Close session. Return `{concepts_updated, next_due_dates}`.

### 5.6 `learner_model_status(filter?: "due"|"mastered"|"shaky"|"all", limit?: 1..50)`

Read-only report. `due` → `review_due_date <= now`. `mastered` → `mastery_score >= 0.8`. `shaky` → `0 < mastery_score < 0.5`. Returns a formatted text block grouped by category with concept, mastery, last_seen, next review.

/\b(teach\s+me|ens[eé][ñn][ae]me|ens[eé][ñn]ale|expl[ií]came\s+(?:desde\s+cero|paso\s+a\s+paso)|quiero\s+aprender|want\s+to\s+learn|walk\s+me\s+through|tutor\s+me\s+on)\b/i

// §B Learner-session verbs
/\b(quiz\s+me|qu[ií]zz?|t[oó]mame\s+(?:un\s+)?quiz|(?:rev[ií]s[aá]|review)\s+(?:de\s+)?hoy|what'?s\s+due|(?:what\s+)?do\s+I\s+need\s+to\s+review|explain\s+back|expl[ií]came\s+de\s+vuelta|mast[eé]r[ií]a|learning\s+plan|plan\s+de\s+aprendizaje)\b/i

## §10 Security / hygiene

- No path handling — pure DB + LLM, no filesystem writes (unlike file-convert/chart tools).
- `plan_id` generated via `randomUUID()` internally; user never supplies arbitrary IDs.
- All tool outputs are strings (per ACI contract).
- Quiz "expected answer" fields are **not** persisted in user-visible transcripts — stored in `learning_sessions.summary` JSON keyed to the grading turn.
- SM-2 `ef` clamp at 1.3 enforced in code AND DB CHECK constraint.
- Learner model concept keys normalized: lowercase + trim + collapse whitespace (prevents "React" vs "react" duplicates).
- NFC-normalize any user-supplied topic string (matches commit 81a161e policy).

## §12 Risks + mitigations

| Risk                                                                 | Mitigation                                                                                                                         |
| -------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| LLM decomposition returns 0 or >8 units                              | `ceiling_units` param + hard clamp + reject-and-retry-once with error prompt                                                       |
| Prerequisite graph is too conservative, blocks progress              | `learning_plan_advance` takes optional `force: true` param; also mastery threshold is 0.7, not 1.0                                 |
| Learner concept key collisions (React vs react, Promise vs promises) | Normalization in `upsertConcept` (lowercase + trim + collapse whitespace + strip trailing `s`)                                     |
| Quiz grading quality depends on LLM judge                            | Rubric-based prompt with 5 explicit criteria; deferred refinement to v7.11.1 if smoke reveals bad calibration                      |
| Scope regex over-fires on "teach" / "review"                         | Anchored with "me" / "today" / "what's due" — dev chatter like "teach the model" lacks these anchors. Tested with 6 negative cases |
