## Scope ŌĆö what v7.5 adds

Six additions (AŌĆōF). Each is isolated, additive, gated.

### A. Failure source classifier (SkillClaw)

**What**: Every experiment gets a `failure_source` label on regression/error: `skill` (the mutation itself is bad), `agent` (LLM misused a good mutation), `env` (infra/timeout/rate-limit). Classifier runs only on `status IN ('regressed','error')` ŌĆö not on pass. Heuristic rules + optional LLM fallback (flag-gated, default off for cost).

**Files**: `src/tuning/failure-classifier.ts` (new, ~120 LOC), `schema.ts` (additive column), `overnight-loop.ts` (call after regression), `types.ts` (enum).

**DB change**: `ALTER TABLE tune_experiments ADD COLUMN failure_source TEXT` ŌĆö additive, idempotent.

**Test**: 12 unit tests covering each heuristic branch + null-on-pass invariant.

### B. `score_child_prop` as default parent selection (HyperAgents)

**What**: Switch `DEFAULT_CONFIG.parentSelection` from `"best"` to `"score_child_prop"`. Already implemented in `parent-selection.ts`; this is a one-line config change + test.

**Why Tier 1**: HyperAgents' report shows `best` over-exploits ŌĆö `score_child_prop` gives novelty-aware exploration (higher scores ŌåÆ less exploitation, less-explored branches get priority). The function is tested; only the default flips.

**Files**: `overnight-loop.ts` (one line), `overnight-loop.test.ts` (1 test asserting default).

**Risk**: **Zero** ŌĆö existing `best` callers keep working via explicit config override. Changed default only affects fresh runs.

### C. Confidence signals in experiment metadata (GEPA Tier 1)

**What**: Optional `confidence_avg` field on experiment result. For now, compute from eval result per-case scores stddev as a proxy. Full logprob threading deferred (requires LLM provider support; Opus/Sonnet expose it, qwen less reliably).

**Files**: `schema.ts` (additive column `confidence_avg REAL`), `eval-runner.ts` (compute stddev proxy), `overnight-loop.ts` (pass through).

**DB change**: `ALTER TABLE tune_experiments ADD COLUMN confidence_avg REAL` ŌĆö additive.

**Test**: 3 tests for stddev proxy computation.

### D. 5-gate monotonic validator (Phantom + Memoria)

**What**: Refactor existing `validateMutation` into a named 5-gate taxonomy with explicit per-gate logging. Adds **safety-keyword preservation gate** (Phantom's conservative editing). Daily cost cap already exists via `CostTracker`; adds **per-tool cooldown** (Memoria pattern ŌĆö block variant that re-mutates the same tool within 24h of a regression).

Gates:

1. **Constitution** ŌĆö hypothesis not overfit to case ID (existing)
2. **Size** ŌĆö mutation Ōēż2x original length (existing)
3. **Worthiness** ŌĆö single-case fixes can't add >20% length (existing)
4. **Safety** ŌĆö NEW: mutation must preserve designated safety keywords (regex list in config; empty by default, populated when a safety keyword exists in the original)
5. **Cooldown** ŌĆö NEW: same (surface, target) can't be re-mutated within 24h if last attempt regressed

**Files**: `src/tuning/gates.ts` (new, ~150 LOC), `overnight-loop.ts` (swap `validateMutation` call for `runGates`), tests.

**Risk**: Mechanical refactor. Existing 3 gates preserved bit-exact; new safety/cooldown gates are additive (both pass by default on fresh cases).

### E. Session trajectory ŌåÆ golden suite promotion (SkillClaw)

**What**: After a variant is activated, harvest the worst regressing cases (from baseline `perCase` where score < 0.5) and optionally promote them to `mined_test_cases` table with source `trajectory`. Gated by threshold: only promote if case was stable-failing across Ōēź3 experiments. Dry-run flag for first N promotions.

**Files**: `src/tuning/trajectory-miner.ts` (new, ~100 LOC), ritual hook.

**Risk**: `mined_test_cases` table already exists and is read by `getActiveTestCases()`. Addition is idempotent.

### F. Inline self-review gate (Superpowers)

**What**: Before a mutation ships to `runEvaluation`, run a synchronous 30s inline self-check ŌĆö same model, 4-item checklist (placeholder scan, type consistency, scope alignment, ambiguity). Zero new model calls for non-mutator path; +1 for each mutation (cheap vs 25-min dispatcher subagent alternative).

**Files**: `src/tuning/self-review.ts` (new, ~80 LOC), `overnight-loop.ts` (one call site before `runEvaluation`).

**Risk**: Adds one short LLM call per experiment. Flag-gated via `ENABLE_SELF_REVIEW` env var, default **off** ŌĆö this is the most uncertain of the six.
