```ts

  options: string | Partial<InfographicOptions>,
```

## §3 Tool interface

```ts
infographic_generate({
  description: string;           // NL or raw DSL ("infographic <template>\ndata\n..."
                                 // or JSON-like options literal starting with "{")
  template?: string;             // optional template name; validated against full 276
  theme?: "light" | "dark" | "hand-drawn";  // default "dark"
  data?: Record<string, unknown>;// structured data; if set, used directly + skips LLM
  output_path?: string;          // absolute, under /tmp or /workspace
  emit?: "render" | "source";    // default "render"
  width?: number;                // optional explicit canvas width
  height?: number;               // optional explicit canvas height
}) → human-readable summary w/ path + bytes + template used
```

Dispatch decision:

1. If `data` is supplied → call `renderToString({ template, data, theme, width, height })` directly. No LLM.
2. Else if `description` starts with `infographic ` (DSL leader) → call `renderToString(description, { theme, width, height })` directly. No LLM.
3. Else → LLM turns `description` into AntV DSL using curated-template hints + theme, then `renderToString(dsl, { theme, width, height })`.

## §4 Security

- No `execFile`, no shell, no network — `renderToString` is pure in-process pure-JS.
- `output_path` absolute + canonical + under `/tmp/` or `/workspace/` only (same model as v7.10, v7.12).
- Description capped at 8000 chars before reaching LLM or renderer.
- SSR has a 10s internal timeout; we add an outer 30s guard via `Promise.race` as belt-and-suspenders.
- `template` param validated against full 276 via `getTemplates()` — rejects typos OR injection attempts (e.g., a template name containing filesystem characters).

## §8 Risks

| Risk                                                                       | Mitigation                                                                                                                              |
| -------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| LLM produces invalid AntV DSL                                              | `renderToString` throws → tool returns error envelope with the raw DSL so caller can inspect. Deferred: retry-with-error-feedback loop. |
| Template updates in 0.2.18+ rename a curated template                      | Curated list is in the tool description (prompt), not enforced. Full-set runtime validation prevents stale references silently passing. |
| AntV's fallback font (`AlibabaPuHuiTi`) renders poorly on non-Chinese text | Per recon, minor — affects edge-case wrap/overlap, not render success. Accept for MVP; upstream tracks via issue #199.                  |
| SSR 10s timeout trips on complex infographics                              | Outer 30s Promise.race gives clearer error surface. If real-world hit rate grows, escalate.                                             |
| Package moves fast (0.2.13 → 0.2.17 in 2.5 months)                         | Pin version in package.json. Bump deliberately via future sprints.                                                                      |
| CLAUDE.md dep-count invariant drift                                        | Update same-commit per INTEGRATION-CHECKLIST.md change-type "new DEPENDENCY".                                                           |

## §10 Deferrals (v7.14.1)

- **PNG output** via the existing v7.1 puppeteer renderer — trigger: first use case where WhatsApp/Telegram re-encodes SVG poorly.
- **Streaming render** (AntV's fault-tolerant progressive-render mode) — trigger: SSE dashboard surface needs live rendering.
- **Template auto-suggest** — LLM picks best template from curated 15 when user omits `template`. Current behavior: LLM picks from 276; curation is prompt-guided.
- **Retry-with-error-feedback** — invalid DSL → re-prompt LLM with error → re-render. Current: one-shot.
- **Custom palette / themeConfig override** — only 3 preset themes at MVP.
- **v7.1 / v7.12 composition** (infographic as video frame, chart inside infographic) — post v7.4 video work.
