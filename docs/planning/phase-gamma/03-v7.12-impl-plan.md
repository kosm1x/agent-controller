# v7.12 — Diagram Generation (`diagram_generate`) Impl Plan

Session 86 · 2026-04-20 · Phase γ S3.

## §0 Scope

One new deferred tool `diagram_generate`. Renders diagrams via apt/npm-installed CLI renderers for declarative DSLs; falls back to an inline LLM-generated SVG+HTML path for free-form architecture sketches.

Reference: `reference_architecture_diagram_generator.md` (Cocoon-AI). The repo is a Claude.ai Skill — no code, only a prompt + palette worth porting (~40 lines).

## §1 Scope decision — 3 formats ship, 2 deferred

Roadmap lists 5 formats: mermaid, d2, plantuml, graphviz, svg_html. **Recon pivot during step 2**: mermaid-cli (`mmdc`) v11 AND v10.9.1 both hang with `ProtocolError: DOM.resolveNode timed out` on this VPS — a puppeteer/Chromium compat issue (4+ minute wall-clock, no output, zero svg produced on both versions). Mermaid deferred to v7.12.1. Shipping 2 formats:

| Format   | Binary         | MVP status | Reason for defer                                                                                                            |
| -------- | -------------- | ---------- | --------------------------------------------------------------------------------------------------------------------------- |
| graphviz | `dot`          | **Ship**   | Apt-installable; covers architecture + ER + state + flowchart topology; <1s render                                          |
| svg_html | inline LLM     | **Ship**   | Zero binary dep; ports the Cocoon palette+spacing prompt                                                                    |
| mermaid  | `mmdc`         | **Defer**  | Puppeteer hang on this VPS (DOM.resolveNode timeout). Revisit when mmdc replaced by Node-API renderer OR upstream fix lands |
| d2       | `d2`           | **Defer**  | Not in apt; requires `curl \| sh` installer (out-of-band). Skip until demand                                                |
| plantuml | `plantuml.jar` | **Defer**  | Requires JRE + `.jar`; ~200MB of Java deps; graphviz covers the UML-ish cases                                               |

v7.12.1 triggers:

- **mermaid**: upstream mermaid-cli DOM.resolveNode fix, OR we switch to direct `mermaid` Node API (`@mermaid-js/mermaid` on Node 22, render via jsdom/happy-dom) bypassing mmdc's puppeteer stack entirely
- **d2**: first user request referring to `d2` specifically, OR graphviz auto-layout proves inadequate for a real workflow
- **plantuml**: first user request for UML sequence-at-scale

## §3 Dispatch table

| format     | Path                                                               | Inputs                                     | Output default |
| ---------- | ------------------------------------------------------------------ | ------------------------------------------ | -------------- |
| `mermaid`  | write DSL to temp `.mmd`, run `mmdc -i in.mmd -o out.svg`          | description (NL or raw mermaid DSL)        | SVG            |
| `graphviz` | write DSL to temp `.dot`, run `dot -Tsvg in.dot -o out.svg`        | description (NL or raw dot DSL)            | SVG            |
| `svg_html` | LLM generates full HTML-wrapped SVG using Cocoon palette + spacing | description (NL; structured hints welcome) | HTML           |

Pipeline when `description` is NL (not DSL): call `infer()` with a dispatch-specific prompt that converts the NL → valid DSL, then render. When `description` already looks like DSL (leading `graph`/`flowchart`/`sequenceDiagram`/`classDiagram` for mermaid, `digraph` for graphviz), skip the LLM step and render directly. Cuts latency + cost when caller already knows the format.

**Supported `diagram_type` values**: `architecture | flowchart | sequence | er | class | state`. These guide the LLM's DSL emission but don't force a specific format — mermaid handles flowchart/sequence/class/state/er natively; graphviz is best for architecture-leaning topology; svg_html is best for free-form "show the system's components and their relationships."

## §4 Tool interface

```ts
diagram_generate({
  description: string;                    // NL describing the diagram, OR raw DSL
  format: "mermaid" | "graphviz" | "svg_html";
  diagram_type?: "architecture" | "flowchart" | "sequence" | "er" | "class" | "state";
  theme?: "dark" | "light";               // default: "dark" (matches Cocoon)
  output_path?: string;                   // absolute path under /tmp or /workspace
  emit?: "render" | "source";             // "source" skips render, returns DSL text only
}) → human-readable summary w/ path + bytes + binary used
```

Param semantics:

- `emit: "source"` — return the DSL without rendering. Useful when caller wants to iterate on the source before committing to a render. Skips `mmdc`/`dot` entirely.
- `theme: "dark"` — threaded into mmdc via `--theme dark`, into graphviz via a default `bgcolor=#0f172a` / `color=#e2e8f0` set of node attrs, into svg_html via the Cocoon palette.

## §5 Security

Same model as v7.10 `file_convert`:

- `execFile(bin, [args...])` — no shell
- `output_path` (if supplied) absolute + canonical + under `/tmp/` or `/workspace/`
- Default output: `/tmp/diagram-<uuid>.<ext>`
- DSL text written to a temp file under `/tmp/`, not executed — mmdc/dot parse; any "injection" lands inside a diagram spec, not argv
- LLM-produced DSL is sandboxed by the renderer's own parser
- 60s timeout per render

No `input_path` whitelist needed — the input is `description` (string), written to a fresh `/tmp` file by the handler. No user-supplied path ever reaches the binary.

## §6 Scope wiring

Create a new `DIAGRAM_TOOLS = ["diagram_generate"]` group — activates on diagram-specific vocabulary. Separate from `specialty` (which bundles chart, research, image generation) to keep prompt-token footprint lean when user wants a simple diagram without pulling in the heavier chart/research tools.

Bilingual regex candidates (EN + ES):

- EN: `diagram`, `flowchart`, `sequence diagram`, `architecture diagram`, `mermaid`, `graphviz`, `dot graph`, `ER diagram`, `class diagram`, `state diagram`
- ES: `diagrama`, `flujo(?:grama)?`, `diagrama de secuencia`, `diagrama de arquitectura`, `diagrama ER`, `diagrama de clases`, `diagrama de estados`

Negative lookahead: `diagramma` (Italian) — unlikely to matter here, but `diagram` is already narrow enough to not need one.

## §8 Risks

| Risk                                                                     | Mitigation                                                                                                                         |
| ------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| `mmdc` bundles its own Puppeteer/Chromium → +250MB install               | Set `PUPPETEER_EXECUTABLE_PATH` to reuse our existing Chromium from playwright. Fallback: accept the dupe.                         |
| mmdc cold start is slow (Chromium boot per render ≈ 3-5s)                | 60s timeout. Doc in description: "first render may be slower."                                                                     |
| Graphviz has many subformats (`-Tsvg` vs `-Tpng` vs `-Tdot`)             | Hardcode `-Tsvg` at MVP. PNG conversion delegated to v7.10 `file_convert` if caller needs it.                                      |
| LLM-produced DSL may be syntactically invalid                            | Render errors surface cleanly; tool returns DSL + error so caller can fix. Optional retry with error feedback deferred to v7.12.1. |
| `svg_html` generation is purely LLM text — could contain XSS if mis-used | Output is a file, not HTML served to a browser from mission-control. Downstream delivery must escape. Not our layer.               |
| Description length bloat — a 10k-char description blows LLM budget       | Cap at 4000 chars in the handler; reject longer with a clear error.                                                                |

## §10 Deferrals (v7.12.1+)

- **d2 format** — trigger = user explicitly requests d2 OR mermaid/graphviz insufficient for a specific case. Install via `curl | sh` is out-of-band.
- **plantuml format** — trigger = sequence diagrams at scale OR specific UML tooling needed. Requires JRE.
- **PNG output** — mmdc can emit PNG directly with `-o out.png`; currently SVG-only at MVP. Trigger = messaging delivery path needs raster.
- **Error-feedback retry loop** — invalid DSL → re-prompt LLM with error → re-render. Currently one-shot. Trigger = observed real-world failure rate >30%.
- **Diagram editing** — take existing source + modification instructions, diff, re-render. Trigger = demand.
- **Mermaid CSS theming** — pass custom `themeVariables`. Current: stock dark + light. Trigger = brand-matched output request.
