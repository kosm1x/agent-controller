## Stage 0 — Reality check (done before plan)

Three things had to be verified against this VPS before the architecture was fixed:

1. **Does Playwright's bundled Chromium survive sequential screenshots?** → **NO.** `chrome-headless-shell` (the bundled binary) crashed on the first `page.screenshot()` after `page.setContent(...)`. Same failure class as v7.1 (SVG pivot) and v7.12 (mmdc hang). Not diagnosable — snap AppArmor + bundled binary interaction.
2. **Does the snap Chromium binary survive sequential CLI screenshots?** → **YES, but slow.** 24 frames × ~1.5s cold-start = 36s. At that rate, a 10-second 24fps video takes 6 minutes. Too slow for interactive use, but does not crash.
3. **Does Playwright launched with `executablePath: '/snap/bin/chromium'` survive sequential screenshots?** → **YES and fast.** 24 frames × ~140ms (warm-loop average) = 3.4 seconds total. Cold start ~500ms, steady state ~100-140ms/frame. This is the architecture.

At 140ms/frame a 10s 24fps video ≈ 34 seconds real time; 30s 24fps ≈ 101 seconds. Acceptable for a backgrounded job.

Chromium writes to `/root/tmp-*` are OK; `/tmp/*` is AppArmor-blocked (v7.4 S1 lesson). Frame dir must be `/root/tmp-video-frames/{jobId}/` not `/tmp/video-frames/...`.

## S1 architecture (MVP — this sprint)

Single new code module + one new tool. Everything reuses existing infrastructure (video_jobs table, composition-protocol sanitization pattern, cinema-prompts-style static catalog).

```
  caller authors HTML under /root/tmp-video-html/*.html
  ↓
  video_html_compose({html_path, fps, width, height, max_duration_sec?})
  ↓
  html-parser.ts — linkedom SSR parse
    - verify path allowlist (/root/tmp-video-html/)
    - extract data-start/data-duration/data-track-index/data-layer from elements
    - compute totalDurationSec = window.__hf.duration() > data-driven max
    - reject if exceeds max_duration_sec or 120s hard cap
  ↓
  html-renderer.ts — Playwright + /snap/bin/chromium
    - launch headless w/ executablePath override
    - newContext({viewport: {width, height}, acceptDownloads: false})
    - page.route('**', block external; allow only file:// under allowlist + data: URIs)
    - page.goto(`file://${htmlPath}`)
    - for each frame t in [0, totalDurationSec] step 1/fps:
        await page.evaluate(t => window.__hf?.seek?.(t), t)
        await page.screenshot({path: `${frameDir}/frame-${N}.png`})
    - browser.close()
  ↓
  ffmpeg concat PNG sequence → MP4 (libx264, yuv420p, fps locked)
  ↓
  rm -rf frameDir (unless KEEP_FRAMES env for debugging)
  ↓
  video_jobs.output_file points at /tmp/video-jobs/{jobId}.mp4
```

## HTML DSL spec

Single `.html` file with these opt-in conventions:

- **Annotation-driven timeline**: any element can carry `data-start` (seconds, float, default 0), `data-duration` (seconds, float, default = totalDurationSec - data-start). Parser computes `totalDurationSec = max(data-start + data-duration)` across annotated elements.
- **Optional `window.__hf` contract**: a `<script>` block may set `window.__hf = { duration: () => number, seek: (t) => void }`. If `duration()` is defined, it overrides the data-driven max. `seek(t)` is called before every screenshot.
- **`data-track-index` / `data-layer`**: informational — ignored by v1 renderer. Preserved for v7.4.3.3 (multi-track compositing).
- **Styling**: CSS animations work (duration-based). GSAP not bundled (deferred to v7.4.3.2).
- **External resources**: blocked by Playwright route handler. Only `file://` inside allowlist + `data:` URIs load. Fonts must be inline `@font-face { src: url(data:...) }`.

Minimal example:

```html
<!doctype html>
<style>
  body {
    background: #111;
    color: #eee;
    font-family: sans-serif;
  }
  #hero {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
    font-size: 72px;
    opacity: 0;
  }
</style>
<div id="hero" data-start="0" data-duration="5">Hola Jarvis</div>
<script>
  window.__hf = {
    duration: () => 5,
    seek: (t) => {
      document.getElementById("hero").style.opacity = Math.min(1, t / 0.5);
    },
  };
</script>
```

## Safety model

1. **HTML path allowlist**: `/root/tmp-video-html/` only. Enforced via `isSafeHtmlPath()` mirroring `isSafeImagePath` from composition-protocol.ts (absolute, no `..`, no shell metachars + null byte, ends `.html`, prefix match).
2. **Network isolation**: `page.route('**/*', route => route.continue({...}) | route.abort())` — only `file://` under allowlist and `data:` URIs proceed. Blocks exfil + external script loads.
3. **JS sandbox limits**: no new privileges — Playwright default context. No geolocation, no downloads (`acceptDownloads: false`).
4. **Total render cap**: 300 seconds wall-clock per job (hard abort via `AbortController` + process kill). Separate from duration cap.
5. **Duration cap**: 120 seconds max composition (matches `video_create`).
6. **Viewport cap**: 1920×1920 (landscape 1920×1080, portrait 1080×1920, square 1080×1080 — conventional).
7. **FPS enum**: `24 | 30 | 60`.
8. **Frame directory**: `/root/tmp-video-frames/{jobId}/` — TTL cleaned by `video_job_cleanup`.
9. **Confirmation**: `videoHtmlComposeTool.requiresConfirmation = true` (matches `video_compose_manifest`).
