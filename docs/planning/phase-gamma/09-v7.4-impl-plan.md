> Session 97 (2026-04-21). Phase γ item 12 (of 13). Augments the existing Phase-β "S5d" video pipeline with hyperframes Tier-1 patterns that do not require a Chromium compositor. Ships S1 (composition engine) + S2a (storyboard + cinema prompts + brand-DNA bridge) this session; S2b (higgsfield/ElevenLabs/lip-sync adapters) deferred to a credential-gated follow-up session, same pattern as v7.3 P4a→P4b.

## QA Audit Plan (2 rounds, per `feedback_audit_iteration.md`)

### Round 1 — focus areas

1. **Command injection on ffmpeg args** — every new arg type-constrained at Zod; attack vector is user-supplied scene text in `-vf "subtitles=..."` filter
2. **TOCTOU on output paths** — two-stage `isUnder(/tmp/video-jobs)` + post-mkdir `realpath` re-canonicalization (per `feedback_v71_chart_rendering` lesson 7)
3. **Temp-file cleanup discipline** — every `throw` path in `runPipeline` has `finally` with best-effort `rmSync` + inner try/catch (per `feedback_v712_diagram_generate` Pattern 4)
4. **Scope regex edges** — 6 pos + 6 FP-neg per new arm (per `feedback_bilingual_scope_regex_patterns`)
5. **LLM prompt injection** in storyboard brief — template substitution single-pass; list/scalar sanitize at storage boundary
6. **Boundary tests for clamps** — `duration` clamp at 15/16/119/120, `manifest_json` at exactly 256KB
7. **ffmpeg timeout enforcement** — `FFMPEG_TIMEOUT_MS=120000` propagates through worker-pool concurrency
8. **Off-by-one in frame quantization** — t=0, t=1/fps, t=1/fps-epsilon, t=duration, t=duration+epsilon

### Round 2 — fix-for-fix regressions

1. Tightened scope regex still matches legitimate ES (`"renderizar el vídeo"`, `"renderizar el video"`, `"hazme una transición de video"`)
2. `ffmpeg_pid` cleared on normal completion — otherwise cancel against a completed job's pid kills unrelated process
3. `quantizeTimeToFrame` preserves existing `overlay-e2e.test.ts` output hash (determinism invariant)

## Why this plan over alternatives

- **Why augment S5d instead of port openmontage/hyperframes wholesale?** S5d ships today. Rewriting working code costs more than augmenting it. The hyperframes patterns we actually need (#1, #2, #3, #5, #7 partial) layer cleanly on top.
- **Why ffmpeg xfade instead of real GL shaders?** Zero deps, already installed, covers 8 of 14 hyperframes shaders by name. The other 6 become v7.4.4 trigger if/when operator says "dissolve looks bad."
- **Why defer Remotion even though Chromium works here?** Roadmap already scopes Remotion as v7.4.3 (competing paradigms shouldn't coexist in v7.4). Chromium path-confinement is flaky enough that a ffmpeg-first architecture is the resilient default.
- **Why split S2a from S2b?** Credential provisioning is its own workflow. Speculative adapter code against unprovisioned keys = dead code. v7.3 P4a→P4b same pattern.
