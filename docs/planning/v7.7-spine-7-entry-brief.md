**Part B — continuity tooling** (Q4 absorption — codify the closure ritual as commands):

- **`close-phase <name>`** — drafts `<NAME>-CLOSURE.md` from the spine table in `<NAME>-GUIDE.md`, drafts the watchlist reframe, drafts an annotated git tag. **DOES NOT push** — operator review gate.
- **`resume <phase>`** — reloads a phase's open queue + watchlist + last-commit ledger into the session brief.
- **`audit-closure <closure-doc>`** — wraps the qa-auditor closure-audit pattern (cross-check spine commits, sum the scoreboard math, verify the anti-mission held, validate memory-pin existence).

**Hard constraint**: the tools MUST NOT execute git ops (commit/tag/push) without operator confirmation. Failure mode to prevent: a typo'd `close-phase` auto-pushing a half-baked closure tag.

**Done when**: v7.7's own closure uses the new commands at least once end-to-end. The drafted `CLOSURE.md` still gets manual editing before the tag — that is the design (draft, not autopilot).
