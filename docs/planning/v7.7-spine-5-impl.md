## §2 — Substrate live (post-Spine-5)

- `self_defining_cohort` table — the ranked identity cohort (`member_id` is a globally-unique prefixed id `<kind>:<raw-id>`). `member_kind` CHECK admits `thread` as documented headroom; v7.7 populates only `project` + `objective`.
- `operator_profile` table — the Q3 skeleton. `written_by` CHECK admits only `'cohort-rollup'`; FK to `self_defining_cohort(member_id)` ON DELETE CASCADE.
- `rollUpCohort()` — deterministic roll-up (no LLM): scores projects by recent `project_log` activity, objectives by NorthStar priority + recency; ranks; takes the top `COHORT_MAX` (30); upserts; deactivates stale members; deletes their `operator_profile` rows; writes the factual profile attributes.
- Daily 05:00 MX cron (`src/cohort/rollup-cron.ts`), `mc-ctl cohort {list,show,rollup}`, `mc_cohort_rollup_total` counter + `mc_cohort_size` gauge.

## §3 — Design decisions & deviations

| Decision                                                                                                                               | Rationale                                                                                                                                                                                                                                                                                                         |
| -------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Cohort stored in a **new `self_defining_cohort` table**, not by extending `jarvis_files.qualifier` (the Conway reference's suggestion) | The `qualifier` CHECK enum cannot be ALTERed without a destructive recreate (a known v8.0-reset batch item). A denormalized table — the GUIDE's alternative route — sidesteps it.                                                                                                                                 |
| Roll-up is **deterministic — no LLM**                                                                                                  | Cohort derivation is substrate; semantic judgment is V8.1/V8.2 capability. Salience is a weighted score over activity counts, priority, and recency.                                                                                                                                                              |
| `operator_profile` write-restricted via a single-value `written_by` CHECK                                                              | Q3 contract: nothing but the roll-up may populate it. `attribute_key`/`attribute_value` are opaque TEXT — V8.2 can redefine value-side semantics with **zero data migration** (anti-mission compliance test ✅).                                                                                                  |
| **Objective filter scoped to `NorthStar/objectives/` + `goals/` + `visions/`** — NOT the whole `NorthStar/` prefix                     | Tightened at closure after live verification: the first roll-up admitted `NorthStar/tasks/` (tactical) and loose `CHANGELOG`/`priority-snapshot` files (namespace metadata) — neither is identity-defining. The `member_kind='objective'` bucket means "a NorthStar identity item" (objective \| goal \| vision). |
| `thread` kind unpopulated                                                                                                              | Only 3 coarse system conversation banks exist — no meaningful per-thread signal. The CHECK admits `thread` as additive-free headroom.                                                                                                                                                                             |

## §7 — Patterns reinforced

- **Verify live, fix what live data reveals** (Spine 4 lesson, 3rd time) — the objective-filter bug was invisible to the Bundle-1 unit tests (synthetic paths) and the auditor (no live NorthStar data); only the live roll-up surfaced it. Closure-stage live verification is a real audit gate.
- **Spec-vs-codebase reconciliation** — the Conway reference's `jarvis_files.qualifier` enum route was un-shippable (un-ALTERable CHECK); used the GUIDE's denormalized-table alternative.
- **Structural enforcement over convention** — the Q3 single-writer contract is a `CHECK` constraint, not a code comment; a test proves it.
