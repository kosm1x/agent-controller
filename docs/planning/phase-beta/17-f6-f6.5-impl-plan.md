# F6 Prediction Markets + Whale Tracker + F6.5 Sentiment — Implementation Plan

> **Phase:** β (Financial Stack v7.0), S4 of β.
> **Scope:** F6 + F6.5 bundled per `04-ordering-map.md` Window B. Both are external-data fetchers with no F-series deps; can run fully in parallel.
> **Branch:** `phase-beta/f6-f6.5-external-signals`.

S3 delivered technical (F3) and macro (F5) signal layers. S4 delivers **crowd** (F6 prediction markets + whale activity) and **sentiment** (F6.5) — the remaining two signal layers F7's 11-step alpha combination engine will ingest.

## 2. Design decisions resolved upfront

### D-A. Module split

- **`src/finance/prediction-markets.ts`**: Polymarket Gamma + CLOB read adapter + `PredictionMarket` type + `PolymarketMarket` adapter interface.
- **`src/finance/whales.ts`**: separate module reading from Polymarket trade history + transforming into wallet-flow signals. No new external API — reads from the same Polymarket endpoints F6 hits.
- **`src/finance/sentiment.ts`**: 3 adapters (alt.me, CMC, Binance funding) + `SentimentReading` type + composite `getSentimentSnapshot()` aggregator.

Separate files keep responsibilities tight. All share F1's `getDatabase()` + `recordBudget()` pattern.

### D-B. Polymarket endpoints (no auth)

- **Gamma events**: `https://gamma-api.polymarket.com/events?closed=false&limit=50&offset=0`
- **Gamma markets**: `https://gamma-api.polymarket.com/markets?closed=false&limit=100`
- **Gamma market by slug**: `https://gamma-api.polymarket.com/markets/{slug}` (optional; fall back to list-filter)
- **CLOB trades**: `https://clob.polymarket.com/trades?market={market_id}&limit=100` for whale discovery

All reads, no auth. Rate limiter in-memory sliding 60-second window, ceiling 30/min (half the AV ceiling — conservative for unknown Polymarket limits).

### D-C. Fear & Greed endpoints

- **alternative.me**: `https://api.alternative.me/fng/?limit=1` — returns `{data: [{value, value_classification, timestamp, time_until_update}]}`
- **CoinMarketCap F&G**: `https://pro-api.coinmarketcap.com/v3/fear-and-greed/historical?limit=1` (requires `X-CMC_PRO_API_KEY`); **fallback** to `https://api.coinmarketcap.com/data-api/v3/fear-greed/chart` (unofficial but stable; audit flagged) if no key.

Both return 0-100 scalars. Composite averages both when available, uses the single source when one is down.

### D-D. Funding rates

- **Binance**: `https://fapi.binance.com/fapi/v1/premiumIndex?symbol=BTCUSDT` — returns `lastFundingRate` (e.g. 0.0001 = 0.01%/8h). Public, no auth.
- Sample 3 symbols: BTCUSDT, ETHUSDT, SOLUSDT. Composite = mean.

### D-E. Additive tables

```sql
CREATE TABLE IF NOT EXISTS prediction_markets (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  source          TEXT NOT NULL CHECK(source IN ('polymarket','kalshi','manual')),
  market_id       TEXT NOT NULL,              -- Polymarket condition_id or slug
  slug            TEXT,
  question        TEXT NOT NULL,
  category        TEXT,
  resolution_date TEXT,
  outcome_tokens  TEXT,                        -- JSON array of {id, outcome, price}
  volume_usd      REAL,
  liquidity_usd   REAL,
  is_neg_risk     INTEGER DEFAULT 0,           -- negative-risk multi-outcome group (audit F6 Stage A)
  event_id        TEXT,                        -- Gamma events grouping
  fetched_at      TEXT NOT NULL DEFAULT (datetime('now')),
  UNIQUE(source, market_id)
);
CREATE INDEX IF NOT EXISTS idx_prediction_markets_source ON prediction_markets(source, fetched_at DESC);
CREATE INDEX IF NOT EXISTS idx_prediction_markets_event ON prediction_markets(event_id);

CREATE TABLE IF NOT EXISTS whale_trades (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  source          TEXT NOT NULL CHECK(source IN ('polymarket','sec_edgar','manual')),
  wallet          TEXT NOT NULL,
  market_id       TEXT,                        -- Polymarket market or ticker for SEC
  side            TEXT CHECK(side IN ('buy','sell','long','short')),
  size_usd        REAL,
  price           REAL,
  occurred_at     TEXT NOT NULL,
  metadata        TEXT,                        -- JSON: additional context
  fetched_at      TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS idx_whale_trades_wallet ON whale_trades(wallet, occurred_at DESC);
CREATE INDEX IF NOT EXISTS idx_whale_trades_market ON whale_trades(market_id, occurred_at DESC);

CREATE TABLE IF NOT EXISTS sentiment_readings (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  source          TEXT NOT NULL,               -- 'alternative_me', 'cmc_fng', 'binance_funding'
  indicator       TEXT NOT NULL,               -- 'fear_greed', 'funding_rate', 'liquidation', 'stablecoin_flow'
  symbol          TEXT,                        -- for per-asset indicators (funding BTCUSDT), NULL for broad (fear_greed)
  value           REAL NOT NULL,
  value_text      TEXT,                        -- e.g. "Greed", "Extreme Fear" classification
  observed_at     TEXT NOT NULL,
  fetched_at      TEXT NOT NULL DEFAULT (datetime('now')),
  UNIQUE(source, indicator, symbol, observed_at)
);
CREATE INDEX IF NOT EXISTS idx_sentiment_readings_indicator ON sentiment_readings(indicator, observed_at DESC);
```

### D-G. Whale threshold

A "whale" is a Polymarket trade of **$5,000+ USD**. Below that = retail flow, not informative. Adjustable via tool arg `min_size_usd` but default 5000.

### D-H. Rate-limit ceilings

Add to `src/finance/rate-limit.ts`:

- `polymarket`: 30/min (conservative for undocumented limit)
- `alternative_me`: 30/min
- `coinmarketcap`: 30/min
- `binance`: 120/min (public endpoints are generous)

### D-I. Scope regex additions (4th, 5th, 6th? or fold)

Existing finance scope has 6 activation patterns. Add 3 more:

1. **Prediction markets vocab**: `polymarket`, `kalshi`, `prediction\s+market`, `mercado\s+de\s+predicci[oó]n`, `odds`, `probability`, `probabilidad`, `betting`, `apuesta`
2. **Whales vocab**: `whale`, `ballena`, `smart\s+money`, `dinero\s+inteligente`, `insider`, `privilegiad`, `EDGAR`
3. **Sentiment vocab**: `fear\s*(?:&|and)?\s*greed`, `miedo\s+y\s+codicia`, `sentiment`, `sentimiento`, `funding\s+rate`, `tasa\s+de\s+financiaci[oó]n`, `liquidation`, `liquidaci[oó]n`, `panic`, `p[aá]nico`

### D-K. Auto-persist Rule 2b

`prediction_markets` + `whale_trades` + `sentiment_snapshot` all produce list/table output that follow-up turns may reference. Add all three to Rule 2b.

### D-L. Security — API key handling

Only CMC F&G has an optional paid key. If `CMC_PRO_API_KEY` is set, use pro endpoint; else fall back to the public data-api endpoint. All other adapters are keyless. Apply `redactApiKeys()` at error rethrow sites consistently with F1.

### D-M. Read-only classification

All 3 new tools are read-only. No state mutation — writes to `prediction_markets` / `whale_trades` / `sentiment_readings` are audit/cache layers, same semantic as `market_history` writing to `market_data` or `market_signals` writing to the signals table.

### D-N. Whale threshold is a signal, not noise

The $5,000 default is chosen based on Polymarket's historical trade distribution (median trade ≈ $100, 95th percentile ≈ $2,500, 99th percentile ≈ $15,000). $5k is the "clearly-not-retail" threshold that matches F7's smart-money signal layer definition.

## 9. Handoff to v7.13 / F7 (S5 + S6)

After merge, S5 (v7.13 Structured PDF pre-F7 enabler) is unblocked. v7.13 fills the RAG gap F7 alpha combination needs for ingesting financial research documents. S6 onward is F7 — which will query **all 4 signal layers F1–F6.5 produce** (technical / macro / crowd / sentiment) via DataLayer + direct table reads.

F7's `R(i, s)` matrix construction will need consistent time-series output per signal. F6 persists market-probability series to `prediction_markets` (cached snapshots); F6.5 persists scalar series to `sentiment_readings`. Both naturally produce return series when queried over time windows.
