# v7.13 Structured PDF Ingestion (Option B, MinerU-free) — Implementation Plan

> **Phase:** β (Financial Stack v7.0), S5 of β.
> **Scope:** Minimum-viable pre-F7 structured PDF ingestion. **MinerU Python service deferred**.
> **Branch:** `phase-beta/v7.13-pdf-structured-ingestion`.

Per V7-ROADMAP v7.13, the full scope includes a MinerU Python service (Docker + systemd + model downloads) for ML-quality table/equation extraction. That's a 1.5-session infrastructure lift. This plan ships the **minimum viable pre-F7 enabler** using existing core deps (`@opendataloader/pdf` → Java-based markdown output) plus pgvector schema enrichment, and defers MinerU to a v7.13-polish follow-up.

**Rationale for Option B**:

1. F7's 11-step alpha combination operates on numeric signal series (F3/F5/F6/F6.5 already shipped). Structured PDF retrieval is an ENHANCEMENT, not a hard F7 blocker.
2. `@opendataloader/pdf` already in core deps. Markdown output renders tables as pipe-format (`| col | col |`) which is trivially detectable.
3. Zero new infrastructure → no Docker, no systemd, no Python, no VPS resource impact, consistent with the zero-new-deps invariant held across S1–S4.
4. If F7/F8 prove retrieval quality is the bottleneck, MinerU can ship as v7.13-polish with data showing the gap.

## 2. Design decisions resolved upfront

### D-A. Modality enum and default

`modality: "text" | "table" | "equation" | "image_caption"`. Default `"text"` for legacy and untagged entries. Column added via additive migration (ALTER TABLE ... ADD COLUMN modality text DEFAULT 'text').

Migration SQL to run on Supabase/pgvector:

```sql
ALTER TABLE kb_entries ADD COLUMN IF NOT EXISTS modality text DEFAULT 'text';
ALTER TABLE kb_entries ADD COLUMN IF NOT EXISTS parent_doc_id uuid;
ALTER TABLE kb_entries ADD COLUMN IF NOT EXISTS section_path jsonb DEFAULT '[]'::jsonb;
ALTER TABLE kb_entries ADD COLUMN IF NOT EXISTS chunk_position int;
CREATE INDEX IF NOT EXISTS idx_kb_entries_modality ON kb_entries(modality) WHERE stale = false;
CREATE INDEX IF NOT EXISTS idx_kb_entries_parent_doc ON kb_entries(parent_doc_id) WHERE parent_doc_id IS NOT NULL;
```

### D-B. Sectionizer strategy — heading-based hierarchy

Markdown from `@opendataloader/pdf` has `#`, `##`, `###` heading levels. Walk the document:

- Track current section stack: array of heading titles at each level
- On new heading: pop stack to the heading's level, push current title
- On content block: emit chunk with `section_path = [...stack]`

Each chunk gets:

- `parent_doc_id`: single UUID for the whole doc (generated at ingest time or user-provided)
- `section_path`: ordered section titles nesting to this chunk
- `chunk_position`: 0-indexed sequential counter within doc

### D-C. Table detection — markdown pipe-format heuristic

`@opendataloader/pdf` renders tables as:

```
| header1 | header2 | header3 |
| ------- | ------- | ------- |
| val A   | val B   | val C   |
| ...
```

Detection: a block of ≥3 consecutive lines starting with `|` and containing ≥2 `|` chars. The 2nd line matches `^\|\s*[-:]+` (separator row). Capture the whole block as a single chunk with `modality: "table"`.

Store the raw markdown in `content` (LLM-readable). Table structure is preserved in the pipe formatting; F7 can parse cells when needed.

### D-D. Chunking size

Text modality: max 1,500 chars per chunk; prefer paragraph boundaries. Empty lines between paragraphs are natural splits. Long paragraphs are sentence-split (regex `/[.!?]\s+(?=[A-Z])/`).

Table modality: one table = one chunk, no splitting (preserves structure). If a table is >8k chars, flag it but don't split — alternatives lose semantic integrity.

### D-E. Embedding generation

Use existing `generateEmbedding()` from `src/memory/embeddings.ts`. Tables get the full markdown as embedding input (caller can later row-serialize at retrieval time for F7 alpha-combination ingestion).

### D-F. Path naming convention

Each chunk's `path` field (the pgvector primary key): `${namespace}/${parentDocBasename}/${sectionSlug}/${position}`.

Example: `finance/apple-10k-2025/MD&A/0003`. Unique per chunk. `namespace` defaults to `"pdf-ingest"` if user doesn't specify.

### D-G. `kb_ingest_pdf_structured` is write-capable

Persists to `kb_entries`. Goes in `WRITE_TOOLS`. Same semantic as `memory_store`.

### D-H. `kb_batch_insert` accepts pre-parsed content

Power-user path for callers that have already extracted structure (Obsidian Drive ingest, scraped data). Input: `entries: Array<{path, title, content, modality?, parent_doc_id?, section_path?, chunk_position?, tags?}>`. Delegates to `pgBatchUpsert` after generating embeddings + `validateKbEntry`.

### D-I. Scope activation

Add 1 new `finance`-group pattern (natural fit since v7.13 was positioned as a pre-F7 enabler): OR create a new `kb_ingest` group. Given the tools are generic KB operations (not finance-specific) and users may want to ingest non-finance PDFs, create a new scope group **`kb_ingest`** so it doesn't couple to the finance regex.

```typescript
export const KB_INGEST_TOOLS = ["kb_ingest_pdf_structured", "kb_batch_insert"];
// Pattern (ES + EN):
/\b(ingest|ingerir|parse|parsear|extract|extraer|import|importar)\b[^\n]{0,30}\b(pdf|document|tables?|tablas?|research|paper)/i;
```

Also activate on direct tool verb: `kb_ingest`, `kb_batch`.

### D-J. Auto-persist Rule 2b

Both new tools return summaries (pages processed, chunks inserted, breakdown by modality). Not follow-up-content. Skip Rule 2b.

### D-K. Error surface

- PDF file not found → friendly "PDF not readable at $path" error
- `extractPdfToMarkdown` throws → caught, formatted with pdf path
- pgvector unavailable → return "pgvector not configured" without crashing
- `validateKbEntry` rejection → include in summary (count rejected chunks)
- Large PDFs → cap at 500 chunks per ingest (configurable via `max_chunks`)

### D-L. `belongs_to` UUID generation

If user doesn't supply `parent_doc_id`, generate one via `crypto.randomUUID()`. Persist the UUID in the ingest result so the user can reference subsequent chunks via it.

## 9. Risks + mitigations

| Risk                                                                         | Mitigation                                                                                                |
| ---------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------- |
| `@opendataloader/pdf` markdown table rendering inconsistent across PDF types | Heuristic requires ≥3 rows + separator line; tolerant of 95%+ PDFs; falls back to text chunk when unclear |
| Large 10-Ks (>1000 pages) → memory pressure                                  | `max_chunks` cap, streaming chunking; document as "for docs <500 pages"; MinerU is the scale answer       |
| pgvector migration forgot on production                                      | Tool checks column existence; degrades to text-only mode with a warning header                            |
| Section stack bug on malformed heading skips                                 | Tested; stack tolerates skips by pushing at current depth                                                 |
| Modality field rejection in validateKbEntry noise filter                     | Tables + equations are high-signal; validate function passes them through unchanged (validated in tests)  |
| Scope false positive on "ingest" in non-PDF context                          | Pattern requires both verb + document-noun; narrow enough                                                 |
