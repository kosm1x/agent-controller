## 2. Design decisions resolved upfront

### D-A. What a "PM signal" is

A PM token is `(market_id, outcome)` — e.g. `(0x...condition_id, "YES")`. Market midpoint in [0, 1] = implied probability. Trading edge = our estimated probability minus market price, on the chosen side. Positive edge on YES = buy YES; negative edge on YES = buy NO (equivalent to 1 − price on NO side).

v1 emits per-token weights; downstream F8.1b adapter translates weights to buys.

### D-B. Simplified 3-feature model (v1)

At each market, compute:

```
p_market  = midpoint (from outcome_tokens[0].price or equivalent)
p_estimate = p_market
             + α · tanh(whale_flow_z)       ← 0 for v1 (no whale data)
             + β · sentiment_tilt(market)    ← small tilt from F&G + funding
```

Constants:

- `α = 0.03` — whale flow weight (conservative; applies only when whale data exists)
- `β = 0.02` — sentiment tilt weight (bounded [-0.02, +0.02])
- `tanh(...)` squashes into [-1, +1] so no single feature dominates

`sentiment_tilt(market)` inspects:

- For crypto-tagged markets (BTC/ETH/crypto in question text): F&G < 25 → +0.02 on "UP" outcomes, > 75 → -0.02. Otherwise 0.
- For non-crypto markets: 0 (v1).

`whale_tilt(market)` (for reference; fires only with data):

- Net whale USD flow into each outcome over last 7 days
- Z-score normalized across all observed markets in the run
- Feeds tanh

### D-C. Kelly-fraction sizing, clipped

For each `(market, outcome)`:

```
edge       = p_estimate - p_market
b          = (1 - p_market) / p_market     ← decimal-odds return if outcome resolves
kelly_raw  = edge / b                       ← simplification of (p·b - q)/b for binary
kelly      = clip(kelly_fraction_scale × kelly_raw, -MAX_WT, +MAX_WT)
weight     = kelly                          ← signed; positive = buy YES side
```

Constants:

- `kelly_fraction_scale = 0.20` — fractional-Kelly (20% of full Kelly; matches academic prudence standard)
- `MAX_WT = 0.02` — ≤ 2% of bankroll per token
- Sum of `|weight|` further clipped at `MAX_TOTAL_EXPOSURE = 0.30` (keep 70% cash reserve)

**Why so conservative**: v1 has no backtest validation, no F7.5 firewall gating the run. Until F8.1c ships, low exposure protects against model error in the wild.

### D-D. Exclusion rules

Skip a market when:

- `|midpoint - 0.5| > 0.45` — prices in [0, 0.05] or [0.95, 1.0] have near-zero edge capacity + extreme slippage
- `liquidity_usd < $1,000` — too thin to fill a meaningful position
- `resolution_date < now + 24h` — about-to-resolve markets collapse on news, not our alpha
- `resolution_date > now + 180 days` — too far out, alpha decay; focus on 1-180d window

Excluded markets appear in `pm_signal_weights` with `excluded=1, exclude_reason='...'`, for audit.

### D-E. Schema — 1 new additive table

```sql
CREATE TABLE IF NOT EXISTS pm_signal_weights (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  run_id          TEXT NOT NULL,                              -- UUID
  run_timestamp   TEXT NOT NULL,                              -- ISO 8601 America/New_York
  market_id       TEXT NOT NULL,                              -- Polymarket condition_id
  slug            TEXT,                                        -- convenience field
  outcome         TEXT NOT NULL,                              -- 'YES' | 'NO' | custom
  token_id        TEXT,                                        -- CLOB token_id; nullable
  market_price    REAL NOT NULL,                              -- midpoint 0..1 at run time
  p_estimate      REAL NOT NULL,                              -- our estimate 0..1
  edge            REAL NOT NULL,                              -- p_estimate - market_price
  whale_flow_usd  REAL,                                        -- signed; null if no data
  sentiment_tilt  REAL NOT NULL DEFAULT 0,                    -- [-0.02, +0.02]
  kelly_raw       REAL NOT NULL,                              -- pre-clip
  weight          REAL NOT NULL,                              -- post-clip, signed
  liquidity_usd   REAL,
  resolution_date TEXT,
  excluded        INTEGER NOT NULL DEFAULT 0 CHECK(excluded IN (0,1)),
  exclude_reason  TEXT,
  created_at      TEXT NOT NULL DEFAULT (datetime('now')),
  UNIQUE(run_id, market_id, outcome)
);
CREATE INDEX IF NOT EXISTS idx_pm_signal_weights_run
  ON pm_signal_weights(run_id);
CREATE INDEX IF NOT EXISTS idx_pm_signal_weights_market
  ON pm_signal_weights(market_id, run_timestamp DESC);
```

## 3. Seed precursor — step 0

Before implementing `pm-alpha.ts`, seed the input tables. One-shot script:

```ts
// scripts/seed-pm-alpha-inputs.ts
import { initDatabase } from "../src/db/index.js";
import { predictionMarketsTool } from "../src/tools/builtin/market.js";
import { sentimentSnapshotTool } from "../src/tools/builtin/market.js";

initDatabase("./data/mc.db");

// 1. Seed prediction markets (ES+crypto+tech categories)
for (const category of ["Crypto", "Politics", "Sports", "Tech"]) {
  await predictionMarketsTool.execute({
    category,
    limit: 20,
    order: "liquidity",
  });
}

// 2. Seed sentiment
await sentimentSnapshotTool.execute({
  indicators: ["fear_greed", "funding_rate"],
});
```

Targets post-seed:

- `prediction_markets` ≥ 50 rows
- `sentiment_readings` ≥ 3 rows (F&G alt.me + F&G CMC + Binance funding)
- `whale_trades` = 0 (acceptable; signal wired but inactive)

**Failure mode**: Polymarket API changes or rate-limits the seed. Script is idempotent via `UNIQUE(source, market_id)` — re-runnable. If seeding fails, the sprint ships without live data but with green tests; smoke is documented as blocked.

## 4. Module contracts

### 4.1 `pm-alpha.ts` — pure

```ts
export interface PmAlphaConfig {
  whaleWeight?: number; // default 0.03
  sentimentWeight?: number; // default 0.02
  kellyScale?: number; // default 0.20
  maxWeightPerToken?: number; // default 0.02
  maxTotalExposure?: number; // default 0.30
  minLiquidityUsd?: number; // default 1000
  minDaysToResolution?: number; // default 1
  maxDaysToResolution?: number; // default 180
  minPriceBound?: number; // default 0.05
  maxPriceBound?: number; // default 0.95
}

export interface PmAlphaInput {
  markets: PredictionMarket[]; // from prediction_markets table
  sentimentReadings: SentimentReading[]; // from sentiment_readings
  whaleTrades: WhaleTrade[]; // from whale_trades (may be empty)
  asOf: string; // ISO timestamp (NY)
  config?: PmAlphaConfig;
}

export interface PmTokenResult {
  marketId: string;
  slug: string | null;
  outcome: string;
  tokenId: string | null;
  marketPrice: number;
  pEstimate: number;
  edge: number;
  whaleFlowUsd: number | null;
  sentimentTilt: number;
  kellyRaw: number;
  weight: number;
  liquidityUsd: number | null;
  resolutionDate: string | null;
  excluded: boolean;
  excludeReason: PmExcludeReason | null;
}

export type PmExcludeReason =
  | "extreme_price"
  | "low_liquidity"
  | "near_resolution"
  | "far_resolution"
  | "stale_market";

export interface PmAlphaResult {
  runId: string;
  runTimestamp: string;
  nMarkets: number;
  nActive: number;
  totalExposure: number; // Σ |weight|
  tokens: PmTokenResult[];
  durationMs: number;
}

export function runPmAlpha(input: PmAlphaInput): PmAlphaResult;
```

**Invariants:**

- All tokens get a row (active + excluded), so audit/explain tools can see why a market was skipped.
- `totalExposure` ≤ `maxTotalExposure`; individual `|weight|` ≤ `maxWeightPerToken`.
- Deterministic given identical inputs (run_id injected by caller for determinism or auto-generated).
- Zero whale_trades → `whaleFlowUsd = null` for every token; `whaleWeight` contribution = 0.

### 4.2 `pm-alpha-persist.ts` — DB-backed

```ts
export function persistPmAlphaRun(result: PmAlphaResult): number; // count inserted
export function readLatestPmAlphaRun(): PmAlphaResult | null;
export function readPmAlphaByRunId(runId: string): PmAlphaResult | null;
```

Transactional batch insert. Same `db.transaction(...)` pattern as F7.5 `alpha-persist`.

### 4.3 Tool handlers

`pm_alpha_run`: loads from F6/F6.5 tables + invokes `runPmAlpha` + `persistPmAlphaRun`. Returns pre-formatted summary.

`pm_alpha_latest`: `readLatestPmAlphaRun()` + pre-formatted output.

## 8. Tool descriptions (ACI)

### `pm_alpha_run`

> Runs the prediction-market alpha pipeline — combines Polymarket midpoint + sentiment tilt + (optional) whale flow into per-token Kelly-fraction weights. Filters out low-liquidity, extreme-price, near-resolution, and far-resolution markets. Persists to `pm_signal_weights`.
>
> USE WHEN operator asks to "run pm alpha", "polymarket alpha weights", "compute prediction-market signals", "mercados predicción", "pondera los mercados de predicción".
> NOT WHEN equity alpha (use `alpha_run`) or paper trading (use `paper_rebalance` — F8.1b).
> Takes ~1-2s. Returns summary with N markets, total_exposure, top 5 weights. Does NOT auto-trade.

Params (Zod): `{ kelly_scale?: number; max_weight?: number; max_exposure?: number; override_config?: string (JSON) }`

### `pm_alpha_latest`

> Returns the most-recent `pm_alpha_run` summary: run_id, timestamp, N markets, active count, total exposure, top 5 tokens by |weight|. No side effects.

Params: none.
