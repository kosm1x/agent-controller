## 0. pm-trader MCP decision (explicit)

V7-ROADMAP §F8.1b listed "pm-trader MCP wrapped as VenueAdapter" as item 1. After implementation review:

- pm-trader is a Python package with 30 tools; stdio MCP server starts in ~660ms; maintains its **own** SQLite DB at `~/.pm-trader/` — violates Jarvis's singleton-DB discipline (CLAUDE.md invariant).
- pm-trader's 657 tests validate a Python implementation we cannot reuse directly.
- Core execution logic (orderbook walking, fill sim) is ~200-400 LOC of portable math per `reference_polymarket_paper_trader.md` — cheap to implement in TS.
- Keeping v7.0 on the "no new language runtimes" + "no new deps" invariants matches the operator's prior stances across 12 β sprints.

**Decision: Do NOT integrate pm-trader MCP at v1.** Build a minimal TS-native `PolymarketPaperAdapter`. The `VenueAdapter` interface stays clean enough that a pm-trader-backed implementation can slot in later if an operator wants it. Skip unblocks progress without cross-language complexity.

Roadmap item "pm-trader MCP registration in mcp-servers.json" is deferred — no open trigger.

## 2. Design decisions resolved upfront

### D-A. Parallel tables, not unified

`paper_portfolio` has shape `(account, symbol, shares, avg_cost)` — equity-only. Extending to `(account, market_id, outcome, shares, avg_cost)` breaks the existing schema. At v1 we ship new parallel tables:

```
paper_balance        → pm_paper_balance
paper_portfolio      → pm_paper_portfolio
paper_fills          → pm_paper_fills
```

Same shape conventions; just keyed by `(market_id, outcome)` instead of `symbol`. F8.1b.3 can unify later with a discriminated column if the operator wants a single-table view.

**Why:** avoids a destructive schema change to `paper_portfolio` (CHECK-constraint addition would require DB reset per CLAUDE.md). Ships faster; audit surface is smaller.

### D-B. Shares vs notional semantics

Polymarket tokens cost `midpoint × shares` USDC to buy YES. One share resolves to $1 if the outcome hits, $0 otherwise. So:

- `avg_cost` = dollars paid per share (≤ 1)
- `market_value` = `shares × current_midpoint` (dollar value at mark)
- `max_profit_if_wins` = `shares × (1 - avg_cost)` (share capped at $1 payoff)
- `max_loss_if_loses` = `shares × avg_cost`

v1 shows `market_value` in output. Profit-to-resolution is available in `pm_paper_history` explain view.

### D-C. Weight → shares math

`pm_alpha_run` emits per-token weights as fractions of bankroll (e.g., `weight = 0.015` = 1.5% of equity). Per-token share count:

```
target_notional = total_equity × |weight|
target_shares   = target_notional / midpoint
side            = weight > 0 ? "buy" : "sell"
```

Shares are rounded to 4dp (matching F8's fractional-share pattern). Minimum notional: $10 per trade (dust threshold).

**Why |weight|:** F8.1a weights are signed. A negative weight on YES means "buy the NO side, which pays when YES doesn't hit". At v1 we simplify: negative weight = sell the YES side (exit existing YES position). Opening a NO position from zero via negative YES weight is deferred to F8.1b.2.

### D-D. Slippage model (v1)

Fixed 20 bps round-trip default (double F8's equity 5 bps — PM spreads are wider). Operator can override via `cost_bps` tool param. Fee formula from `reference_polymarket_paper_trader.md` is deferred to F8.1b.2 along with orderbook walking; fee and slippage are bundled into the flat 20 bps at v1.

**Why 20:** Polymarket CLOB typical spread at mid-liquidity is 50-150 bps per side; 20 bps round-trip understates but is a reasonable placeholder until orderbook-aware simulation ships. Errs on the permissive side (strategy looks better than reality) — deliberately so at v1 so we can see whether the alpha fires at all.

### D-E. Ship-gate semantics: NONE at v1

`pm_paper_rebalance` checks only:

- `pm_alpha_latest` exists and has ≥ 1 active weight → proceed
- Otherwise → clear error message

Unlike F8 `paper_rebalance` (which reads F7.5 `ship_blocked`), v1 has no firewall. Operator MUST explicitly call `pm_paper_rebalance` — no auto-fire via ritual. Firewall is F8.1c.

### D-F. Starting cash: $10K (10% of equity default)

F8 paper_balance starts at $100K USD. PM paper starts at **$10K USDC** — conservative default reflecting PM's higher risk-per-dollar vs equity. Operator can override on first run via `initial_cash` param (same pattern as F8).

### D-G. Position discriminated union

Extend existing `Position`:

```ts
// Existing (F8 equity)
export interface EquityPosition {
  kind: "equity";
  symbol: string;
  shares: number;
  avgCost: number;
  marketValue: number;
  unrealizedPnl: number;
  stale: boolean;
}

// New (F8.1b PM)
export interface PolymarketPosition {
  kind: "polymarket";
  marketId: string;
  outcome: string; // "Yes" | "No" | custom
  slug: string | null;
  tokenId: string | null;
  shares: number;
  avgCost: number; // USDC per share
  marketValue: number; // shares × current midpoint
  unrealizedPnl: number;
  stale: boolean;
}

export type Position = EquityPosition | PolymarketPosition;
```

Backwards compat: existing `Position` references resolve to the union. Callers using `p.symbol` break at compile time — **this is desired**; forces explicit handling of the cross-venue case. Migrate `PaperEquityAdapter` callers to assert `if (p.kind === "equity")` or set `kind: "equity"` in the existing adapter.

### D-H. Schema — 3 new tables

```sql
CREATE TABLE IF NOT EXISTS pm_paper_balance (
  account         TEXT PRIMARY KEY DEFAULT 'default',
  cash_usdc       REAL NOT NULL DEFAULT 10000,
  initial_cash    REAL NOT NULL DEFAULT 10000,
  last_updated    TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS pm_paper_portfolio (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  account         TEXT NOT NULL DEFAULT 'default',
  market_id       TEXT NOT NULL,
  outcome         TEXT NOT NULL,
  token_id        TEXT,
  slug            TEXT,
  shares          REAL NOT NULL,                              -- 4dp fractional
  avg_cost        REAL NOT NULL,                              -- USDC / share
  opened_at       TEXT NOT NULL,
  last_updated    TEXT NOT NULL DEFAULT (datetime('now')),
  UNIQUE(account, market_id, outcome)
);

CREATE TABLE IF NOT EXISTS pm_paper_fills (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  fill_id         TEXT NOT NULL UNIQUE,                       -- UUID
  thesis_id       INTEGER,                                    -- logical FK to trade_theses
  account         TEXT NOT NULL DEFAULT 'default',
  market_id       TEXT NOT NULL,
  outcome         TEXT NOT NULL,
  token_id        TEXT,
  side            TEXT NOT NULL CHECK(side IN ('buy','sell')),
  shares          REAL NOT NULL,
  fill_price      REAL NOT NULL,                              -- midpoint ± slippage
  gross_notional  REAL NOT NULL,
  slippage_bps    REAL NOT NULL DEFAULT 0,
  realized_pnl    REAL,                                        -- non-null on sells
  filled_at       TEXT NOT NULL,
  created_at      TEXT NOT NULL DEFAULT (datetime('now'))
);
```

Additive; live-applicable.

## 3. Module contracts

### 3.1 `src/finance/pm-paper-adapter.ts`

```ts
export class PolymarketPaperAdapter implements VenueAdapter {
  name = "polymarket_paper" as const;
  clock: Clock;

  constructor(opts: PolymarketPaperAdapterOpts) { ... }

  async getMarketData(symbol: string): Promise<MarketQuote>;
  async placeOrder(order: Order): Promise<OrderResult>;
  async getPositions(): Promise<Position[]>;  // returns PolymarketPosition[]
  async getBalance(): Promise<Balance>;
  async getFills(opts?: GetFillsOpts): Promise<Fill[]>;
}
```

`symbol` param for `getMarketData` is `{market_id}:{outcome}` to avoid overloading the `Symbol` type. `PaperEquityAdapter.getMarketData` uses bare `symbol` — OK since the two adapters never share a caller.

`getMarketData` reads from `prediction_markets.outcome_tokens` JSON. Cache hit = return. Stale (>5d since `fetched_at`) = throw → caller rejects order.

`placeOrder`:

1. Validate order type === "market", side ∈ {buy, sell}, quantity > 0
2. Fetch current midpoint via `getMarketData(marketId:outcome)`
3. `fill_price = midpoint × (1 + slip_bps/10000)` for buy, `-` for sell
4. `gross_notional = quantity × fill_price`
5. Pre-flight: insufficient cash → reject; short-sell exceeds held → reject
6. `db.transaction(...)` — insert pm_paper_fills + upsert pm_paper_portfolio (weighted-avg cost on buy, FIFO realized P&L on sell) + update pm_paper_balance cash

Same pattern as F8's PaperEquityAdapter.

### 3.2 `src/finance/pm-paper-executor.ts`

```ts
export async function runPmRebalance(
  opts: PmRebalanceOpts,
): Promise<PmRebalanceSummary>;
```

Steps:

1. Read `pm_alpha_latest` → get active weights (excluded=0)
2. Get current portfolio: `adapter.getPositions()`
3. For each active token:
   - `target_notional = total_equity × |weight|`
   - `target_shares = target_notional / current_midpoint`
   - `delta = target_shares - current_shares`
   - If `|delta × midpoint| < DUST_MIN ($10)`, skip (no micro-trades)
   - Generate `Order`: side based on sign(delta)
4. Sort orders: sells first (free cash), buys after
5. Execute serially via `adapter.placeOrder`
6. Persist thesis row `trade_theses.entry_signal = 'pm_weekly_rebalance'`
7. Back-link fills to thesis by fill_id

Same shape as F8's `paper-executor.ts`.

### 3.3 `src/finance/pm-paper-persist.ts`

Mirrors F8's `paper-persist.ts`:

- `DEFAULT_PM_ACCOUNT = 'default'`
- `DEFAULT_PM_INITIAL_CASH = 10_000`
- `initAccount(account, cash?)` / `readBalance(account)` / `updateCash(account, cash, nowIso)`
- `readPortfolio(account)` / `readPosition(account, marketId, outcome)`
- `applyBuyToPortfolio(...)` / `applySellToPortfolio(...)`
- `insertFill(...)` / `readFills(opts)`
- `linkFillsToThesis(thesisId, fillIds[])`

## 8. Tool descriptions (ACI)

### `pm_paper_rebalance`

> Executes a Polymarket paper-trading rebalance from the latest `pm_alpha_run` weights. Reads `pm_alpha_latest`, sizes each active token as `|weight| × total_equity / midpoint`, generates MARKET orders, executes via synthetic fills at midpoint × (1 ± slip_bps/10000), persists to `pm_paper_fills` + `pm_paper_portfolio`. Takes ~0.5-1s.
>
> USE WHEN operator asks to "rebalance polymarket positions", "ejecuta pm paper trade", "rotar polymarket", "aplica pesos de polymarket".
> NOT WHEN equity rebalance (use `paper_rebalance` from F8) or analysis only (use `pm_alpha_run`/`pm_alpha_latest`).
>
> Does NOT auto-trade — operator-explicit only. No F7.5 firewall at v1 (deferred to F8.1c). Starts with $10K USDC paper cash on first run (configurable via `initial_cash`).

Params: `{ cost_bps?: number; initial_cash?: number; notes?: string; force?: boolean }`

### `pm_paper_portfolio`

> Current Polymarket paper portfolio: positions (market_id + outcome + shares + avg_cost + market_value + unr P&L), USDC cash, total equity. Read-only, ~100ms.

Params: none.

### `pm_paper_history`

> Recent Polymarket paper fills. Filter by market_id / outcome / since / limit. Default limit 20. Read-only.

Params: `{ market_id?: string; outcome?: string; since?: string; limit?: number }`
