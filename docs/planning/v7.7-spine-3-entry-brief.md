## What this spine is

**S5 — Skills-as-Stored-Procedures** — graduate skills from prose recipes into versioned procedures stored as files with frontmatter (`name`, `description`, `inputs`, `tests`), registered into the tool source manager at boot, and runnable via `skill_run(name, args)`.

**Spec** (READ FIRST): `docs/planning/v8-substrate-s5-spec.md` (561 lines, more substantive than S2/S3 specs). Reading order per its preamble: §1 problem → §2 current state → §3 precedents → §4 frontmatter → §5 storage → §6 retrieval → §7 lifecycle → §8 critic gate → §9 test harness → §10 failure handling → §11 toolsource integration → §15 open Qs (6 open, none blocking).

**Estimate**: ~10 days, 5 phases. Largest substrate item by effort.

## Phase 1 — Schema + frontmatter parser (~2 days)

Per spec §13 Phase 1:

**Goal**: Add 1 schema migration (additive) + parse YAML frontmatter from `jarvis_files` `skills/*` paths.

**Concrete tasks**:

1. **Schema**: `skill_versions` + `skill_active` + `skill_test_results` tables per spec §5. Inline in `src/db/index.ts` (matches Spine 1 P1 + Spine 2 B1 pattern).
2. **Parser**: `src/skills/frontmatter.ts` — parse YAML frontmatter from a `jarvis_files` path's contents. Fields per spec §4: `name`, `description`, `inputs[]`, `tests?`, `outputs?`, optional `body_template`. Validation via Zod (mirrors S2 schema pattern).
3. **Loader**: read all `skills/*.md` from `jarvis_files`, parse frontmatter, upsert into `skill_versions` table with `created_by='boot-scan'`.
4. **Tests**: schema migration + frontmatter parser (valid, invalid, missing fields) + loader idempotency.
5. **qa-auditor** pass on the Phase 1 bundle.

**Done when**: All `jarvis_files/skills/*` files are parsed and registered in `skill_versions` table at boot.

## Phase 2 — Critic + test harness (~3 days)

Per spec §8 + §9: critic gates skill activation. Borrows Voyager's critic-as-write-gate pattern (same primitive applied in Spine 1 P2a + P2b).

**Concrete tasks**:

1. `src/skills/critic.ts` — second LLM call grades a skill's `body_template` against its declared `tests`. Frozen system prompt (mirror Spine 1's critic.ts shape).
2. `src/skills/test-harness.ts` — runs declared tests against the skill (table-driven; each test = input + expected output stub). Output: pass/fail count per skill version.
3. `skill_test_results` table populates per run.
4. On critic pass + N tests pass: skill version becomes `active=1` in `skill_active` table.
5. Tests for critic mock + harness.
6. **qa-auditor** pass.

**Done when**: A new skill goes from frontmatter file → parsed → critic-gated → tested → activated (or rejected with reason logged in `skill_test_results`).

## Phase 3 — Vector retrieval (~2 days)

Per spec §6: embed skill `description` field; retrieval is cosine-similarity over the embedding column.

**Concrete tasks**:

1. Add `description_embedding BLOB` column to `skill_versions` (additive ALTER — embedding-bytes; mc.db pattern).
2. Backfill via `qwen3-embedding-coder` (1024d, same as Hindsight) — open Q1 in spec defers to Phase 3 measurement.
3. `src/skills/retrieval.ts` — `searchSkills(query, topN)` returns ranked active skills.
4. Tests with synthetic embeddings.
5. **qa-auditor** pass.

**Done when**: `searchSkills("compose HTML")` returns the HTML-compose suite in top-3.

## Phase 4 — Callable-tools mode (~2 days)

Per spec §11: `skill_run(name, args)` invocable as a builtin tool. ToolSourceManager registers active skills as virtual tools.

**Concrete tasks**:

1. `SkillsToolSource` adds `skill_run` as a meta-tool; the source manager dispatches by `name` arg.
2. Scripts execution: per spec §15 open Q2, default to nanoclaw isolation (NOT inline; security risk).
3. Cross-skill composition: skill A can invoke skill B via skill_run from within L2 body. Cycle detection + depth bound (max 3) per spec §15 Q4.
4. Tests.
5. **qa-auditor** pass.

**Done when**: `skill_run("compose-html-dark", {...})` produces output via the registered skill.

## Phase 5 — Operator surface + activation gate (~1 day)

**Done when**: Operator can list / inspect / activate skills via CLI + HTTP; activation gate met; Spine 3 marked CLOSED in V7.7-GUIDE.
