## Mission

Ship V8.1 — the **Proactive Context Engine**. The shift V8-VISION §4-V8.1 names: from a once-daily ritual that _summarizes data_ into a continuous reflection process that _constructs judgments_. The morning brief becomes one surface of that engine — "this is at risk, this has momentum, this is today's highest-leverage move" — not "here is what happened."

Orienting metric: **the operator stops needing to ask "what should I be looking at?"** — the brief already said it. Activation gate (§13 below): cache-read ratio ≥80% over 24h with brief generation included, promote-rate ≥60% on the morning surface within 30 days.

## The seven phases

V8.1 phases are **sequential** (unlike v7.7's parallel spines) — each consumes the prior. Spec §12 is the authoritative per-phase design; this guide is the operating layer.

### Phase A — Substrate reconciliation (~1-2 days)

Confirm the v7.7-shipped Conway substrate is V8.1-ready and close the gaps the spec §12 phases assumed:

1. **Cohort → prompt wiring** — ✅ **DONE** (2026-05-20). The self-defining cohort is surfaced in `buildJarvisSystemPrompt`'s stable (P2) section via a new pure `cohortSection()` — Conway Pattern 2's grounding payoff. **Security:** the cohort is operator-private (business names, health objectives), so `cohortSection` takes a required `ownerChannel` param (default-deny) and the router gates it with `isOwnerChannel()` — community-manager/public channels get nothing. R1 audit caught the leak as a Critical; R2 verified the fix. Live-verified: owner channel renders 26 members, public channel renders `""`.
2. **`recall_audit` dormancy** (`S6-recall-audit-dormant`) — ✅ **DONE** (bundle `V8.1-PA-recall-logging`, 2026-05-20). Decision: restore `logRecall` — wired into `SqliteMemoryBackend`'s primary path via an `instrument` constructor flag (not `HINDSIGHT_ENABLED=true`, which would reverse the queue #15 demote verdict). `recall_audit` receives fresh `source='sqlite-primary'` rows again; live-verified post-deploy. **Remaining sub-item 2b**: the Spine 6 coherence-drift signal stays `enabled=0` until its baseline is recomputed from ~7d of fresh rows (the old 0.0406 was Hindsight-era) — then flip `enabled=1`. Scheduled: routine `trig_012BrPZ5VGXZPaqcPW6f8vt4`, fires 2026-05-27.
3. **General-events seed sufficiency** — ✅ **DECIDED** (2026-05-20, accepted-with-rationale). Live count: 12 `general_events` rows, all `created_by='seed'` — thin vs the spec's 30-50. Decision: **do NOT speculatively bulk-seed.** Hand-authored seed events are guesses; the designed population path is auto-discovery (`S4-B1-W4` — nightly LLM clustering of episodic memory into general events), which produces events derived from real data. Auto-discovery is **NOT in Phase A scope** (it is a capability bundle, not reconciliation) but is **promoted to a Phase 6 prerequisite**: it must land before Phase 6's briefing retrieval, OR Phase 6 entry re-evaluates seed sufficiency against the then-concrete `retrieveForBriefing` shape. 12 seed events is accepted for Phases 4-5 (which do not consume briefing retrieval).

**Done when**: the three gaps are closed or explicitly accepted-with-rationale, and `retrieveForBriefing` returns a usable cohort + general-events set. ✅ **All three closed** — 1 + 2 shipped, 3 decided. Phase A complete.

## Anti-mission — what V8.1 will NOT do

- **No V8.2 / V8.3 work.** V8.1 is the brief; V8.2 is strategic proposals; V8.3 is autonomous execution. If a task drifts toward "the agent proposes an initiative" or "the agent acts," it is V8.2/V8.3 — defer.
- **No new runners.** The reflection runner is the fast-runner with a reflection-specific prompt + config (Letta same-class pattern) — not a 6th runner.
- **No memory-architecture redesign.** pgvector + the v7.7 Conway substrate are the memory layer. Compose against them.
- **The brief constructs judgments, it does not just summarize.** A brief that reads as "here is what happened" has failed the mission — `noted`-posture >60% is the §13 watchpoint that says so.
- **No autopilot.** Briefings are proposals (`proposed_briefings` with promote/discard); the operator's first morning interaction promotes or discards. Reversibility from day one.

## Operating rules

All v7.7 rules carry forward. Especially load-bearing for V8.1:

1. **Audit every bundle** — qa-auditor R1 per ship; R2 on Critical-bearing/contract-drift bundles. v7.7's ~33% real-bug rate held across ~24 audits.
2. **Verify auditor empirical claims** — reinforced ~7× across v7.7. An auditor's "X does not exist" needs checking against the authoritative source (git, the live DB), not a proxy.
3. **Verify live before claiming done** — deploy + run it + read the output. Three v7.7 spines shipped a bug only the first live run exposed.
4. **Spec-vs-codebase reconciliation** — Phase A IS this rule applied to V8.1's own substrate. The spec is design intent; verify against what v7.7 actually shipped.
5. **Construct, do not summarize** — the mission discriminator. Every briefing-related bundle is judged against it.

## Activation gate & measurement

Per spec §13 / V8-VISION §4-V8.1:

- **Cache-read ratio ≥80%** over a 24h window with brief generation included.
- **Promote-rate ≥60%** on the morning surface within 30 days.
- **Audit-cycle rate → 0** within 60 days (the S2 critic should make "Audited?" unnecessary).
- Watchpoints: `noted`-posture >60% (summarizing not judging), promote-rate <30% (briefs not useful), cache <70% (cohort/general-events sizing).
