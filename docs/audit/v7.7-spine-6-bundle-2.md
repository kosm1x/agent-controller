## The data-source dormancy finding

Bundle 2 recon surfaced that `recall_audit` has **1460 rows but none since 2026-05-10**. Root cause (pre-existing, not Spine 6's): the Hindsight demote (queue #15) routes recall through `SqliteMemoryBackend`, which does not call `logRecall`/`applyOutcomeBias` — only `HindsightMemoryBackend` does, and `HINDSIGHT_ENABLED=false`. So `recall_audit` logging and the coherence outcome filter are both dormant.

**Design response** — the `recall_coherence_suppression_rate` signal is seeded **`enabled=0` (disabled-pending)** with `baseline_query: "awaiting:recall-audit-active"`, mirroring Spine 2's exact pattern for the 10/13 signals whose substrates had not shipped. The real activation SQL lives in `notes`; the baseline `0.0406` is real (measured from the 1460 historical rows). Re-architecting the recall backend to restore logging is **out of Spine 6's substrate scope** — it would change live recall behaviour and touches the deliberate Hindsight-demote routing. Queued instead (`S6-recall-audit-dormant`).
