## Tier-1 → R1-only downgrade rationale

R2 catches "bugs INSIDE R1 fixes" + adjacent surfaces. Post-R1 surface examination:

- **C1 fix** (disable WhatsApp counter signal): the fix is operational (flip enabled=0 + replace baseline_query with sentinel). No code path changed; no new bug surface. Trigger to re-enable is documented + queued.
- **W3 fix** (disable schema_migration_drift): same operational shape — flip enabled=0 + sentinel + notes. No code change.
- **W5 fix** (shutdown wiring): one new line in shutdown handler, dynamic import for boot-order independence. Mirrors the surrounding pattern (stopRitualScheduler etc.). Low surface for follow-up bugs.
- **W1, W2, I1, I4 fixes**: documentation/comment changes, no logic.
- **I3 fix**: added integration test that ALREADY caught its target bug (cost_usd typo). Test pinning is locked.

R2 would have spent ~30-60 min finding ~0 new issues. The fix shape was "disable signal" or "add comment" or "add test that catches the class" — none introduce new failure surface.
