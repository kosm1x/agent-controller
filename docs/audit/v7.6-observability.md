## Bug detail — #1 router.ts:437 `fullCount` drift

### Symptom

```typescript
const fullCount =
  CORE_TOOLS.length +
  SCHEDULE_TOOLS.length +
  MISC_TOOLS.length +
  BROWSER_TOOLS.length +
  CODING_TOOLS.length +
  SPECIALTY_TOOLS.length +
  RESEARCH_TOOLS.length +
  (process.env.GOOGLE_CLIENT_ID ? GOOGLE_TOOLS.length : 0) +
  (process.env.WP_SITES ? WORDPRESS_TOOLS.length : 0) +
  2; // memory
console.log(`[router] Tool scope: ${tools.length}/${fullCount} tools`);
```

### Drift class

Hand-rolled denominator that is not derived from the same source-of-truth that the producer (`scopeToolsForMessage` in `scope.ts:1144-1285`) uses. When a new tool group is added — FINANCE / SEO / VIDEO / SOCIAL / ADS / ALPHA / BACKTEST / PAPER / DIAGRAM / CHART / TEACHING / GRAPH / XPOZ / KB_INGEST / INTEL / UTILITY / BROWSER_EXTRA / CRM_TOOLS_SCOPE / MARKET_RITUAL / PM_ALPHA / PM_PAPER — the producer immediately starts emitting that group's tools, but the denominator stays stale forever. **22 groups missing from fullCount**, plus `2` instead of `3` for memory tools.

### User-visible failure mode

The scope-fraction observability metric (`tools.length / fullCount`) becomes meaningless within weeks of any new tool group landing. Because `tools.length` can exceed `fullCount`, the "X/Y tools loaded" log becomes a numerator-greater-than-denominator nonsense for meta-scope expansions. Operator can't tell from the log whether a particular request loaded few-vs-many tools relative to the universe.

### Fix shape

`scope.ts` now exports `getAllAvailableTools(options: ScopeOptions): Set<string>` — single source of truth co-located with `scopeToolsForMessage`. Returns the universe of tool names that the producer could ever emit for the given env config. `router.ts:437` now reads `fullCount = getAllAvailableTools(opts).size`.

Invariant pinned by tests: `tools.length <= getAllAvailableTools(opts).size` for any input.

### Coupling tests added (`scope.test.ts` describe block "getAllAvailableTools — fullCount producer/consumer coupling")

1. Envelope contains every tool a meta-scope expansion emits
2. `tools.length` never exceeds `universe.size` for any of 14 sample messages spanning every domain
3. Env-conditional Google extends the universe only when `hasGoogle=true`
4. Memory off-by-one regression guard — universe grows by exactly 3 (not 2) when `hasMemory` flips
5. CRM tools present when `hasCrm=true`
6. WordPress + Google + CRM env gates each contribute distinct tools
7. **Syntactic regression guard**: the universe MUST contain `VIDEO_TOOLS / SEO_TOOLS / ADS_TOOLS / CHART_TOOLS / TEACHING_TOOLS / CRM_TOOLS_SCOPE` — if anyone shrinks the helper back to the 9-group form, this test trips.

## Round 1 — qa-auditor (PASS WITH WARNINGS)

Audit confirmed:

- 100% coverage parity between `getAllAvailableTools` and `scopeToolsForMessage` (every conditional add in the producer present in the universe)
- No over-counting (every universe member is reachable from some producer path)
- No parallel arithmetic remaining (searched `_TOOLS\.length` and `fullCount` across `src/` — only matches are inside the test file's intentional sentinel asserts and the new call site)
- Stale named imports (`CORE_TOOLS`, `SCHEDULE_TOOLS`, ...) at router.ts:101-115 cleanly removed; no other usage in router.ts

Findings:

- **W1** — `tools.length ≤ universe.size` invariant only ran with `ALL_ON`. A producer that pushed a gated tool while the gate was off would slip through. **Fixed**: invariant now runs as `it.each` over 6 env-variants (`ALL_ON`, each gate flipped off, `ALL_OFF`).
- **W2** — Meta-scope envelope test only ran with `ALL_ON`, masking gate-omission bugs. **Fixed**: same `it.each` over 6 env-variants applied to the envelope test.
- **S1** — Sentinel set was biased toward groups the OLD arithmetic omitted, not toward groups most likely to be the NEXT drift target. **Fixed**: sentinel now covers two axes — historical drift (VIDEO/SEO/ADS/CHART/TEACHING/CRM) AND recent additions (PM_ALPHA/PM_PAPER/MARKET_RITUAL/KB_INGEST/GRAPH/XPOZ).
- **S2** — Gate-arithmetic test used a loose `> min.size + 5` threshold that could mask a missing gate add. **Fixed**: now asserts exact delta (`expectedDelta = GOOGLE_TOOLS.length + WORDPRESS_TOOLS.length + CRM_TOOLS_SCOPE.length + 3`) so any forgotten gate trips immediately.

Round-1 finds: 0 critical / 2 warnings / 2 suggestions / 0 bundle-regressions. All 4 addressed in the same commit (no separate fix commit needed; the bundle hadn't shipped yet).

## Lessons

1. **Hand-rolled denominators rot the moment a new producer appears.** Counter consumers must be derived from the same source-of-truth function the producer uses, not enumerated by hand. The fix isn't "remember to update the arithmetic" — it's "delete the arithmetic and call a helper."

2. **The "obvious" non-drift sites still earn one minute of audit each.** All 8 non-drift candidates were verified in <1m by reading the function and confirming producer/consumer share the data structure at-emit. The audit cost is small; the false-confidence cost of skipping is large.

3. **Off-by-one in addends survives all unit tests.** The `2 // memory` line was wrong (3 memory tools, not 2) and no test caught it for as long as it existed because tests on `scope.ts` never read the call site's arithmetic. Coupling tests must include the actual call-site invariant, not just the producer's properties.

4. **Migration scripts are non-drift by structure.** When the producer (loop bound) and consumer (loop body) are in the same function, the counter cannot drift between releases. Document this disposition explicitly so future audits don't re-classify them.
