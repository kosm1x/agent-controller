## Tier-1 → R1-only downgrade rationale

R2 promised in Bundle 2's audit log because Bundle 3 "touches drift_alerts.delivery_status + adds P0 push + adds suppression API — three new state-mutation surfaces." Post-R1 surface examination:

- **C1 fix** (broadcastToAll callback signature): the callback path is narrow (3 lines added to router.ts, ~50 lines of counter-bumping in push.ts). The regression test exercises the callback signature end-to-end. No new failure surface; the callback can't throw (try/catch in router.ts internal).
- **W1-W4 fixes** (4 Warnings): all small. W2 (placeholderForRow) replaced one Set-construction with another. W3 (heading) one-line conditional. W4 (case-insensitive) `.toLowerCase()` normalize. None introduce new logic surface to sweep.
- **I3 fix**: net-new regression test, no production code change.

R2 would have ~30-60 min finding ~0 new issues. The DEFER bucket items (R1-I1/I2/I4/I5) are all either edge-case hardening or future-design concerns; none invalidate Bundle 3's correctness.

Where R2 WOULD have been mandatory: if Bundle 3 had added auto-unsuppress (it didn't — deferred to v8.0) OR if push went directly to operator chat without the spec's broadcastToAll surface (it didn't).
