## Closure-stage fix — objective filter

`rollUpCohort`'s `scoreObjectives` originally selected `jarvis_files WHERE
path LIKE 'NorthStar/%'`. The first **live** roll-up after deploy (35
candidates → 30 active) surfaced that this admitted `NorthStar/CHANGELOG-*.md`
and `priority-snapshot-*.md` (namespace metadata) and `NorthStar/tasks/*`
(tactical, episodic) as cohort "objectives" — none are identity-defining,
and the Conway Pattern 2 cohort is precisely the identity layer.

**Fix**: scope the filter to the genuine identity subdirs —
`NorthStar/objectives/`, `NorthStar/goals/`, `NorthStar/visions/`. The
post-fix roll-up: 26 candidates (17 projects + 9 identity objectives), a
clean cohort, no metadata files. Bundle-1 unit tests' synthetic objective
paths were moved under `NorthStar/objectives/` accordingly; all 19 cohort
tests pass.

This fix was invisible to the Bundle-1 unit tests (synthetic paths) and to
the Bundle-1 auditor (no live NorthStar data) — only the live roll-up
exposed it. It is the 3rd reinforcement of the Spine-4 "verify live, fix
what live data reveals" lesson.

## R1 / R2 decision

**No separate R1.** The fix is a single SQL `WHERE`-clause tightening plus
mechanical test-path updates, fully covered by the (passing) Bundle-1 test
suite and the live roll-up verification in `v7.7-spine-5-impl.md` §5. Per
the Spine 3/4 cadence, R1 is per code-bundle and R2 is reserved for
Critical-bearing/contract-drift bundles; a closure-stage one-clause fix with
green tests + live confirmation does not warrant a fresh agent pass. The
full suite runs at the pre-commit hook.
