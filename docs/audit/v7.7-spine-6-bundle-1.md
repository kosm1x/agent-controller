## Design notes

- **Additive, not a rename.** `recallMode` is a new high-level option; the legacy `excludeOutcomes`/`includeFailed` knobs still work and take precedence. Behaviour is unchanged for every caller that does not set `recallMode` (and there are zero production callers of the legacy knobs today).
- The exclude-set precedence in `resolveExcludeOutcomes`: `includeFailed` → explicit `excludeOutcomes` → `recallMode` → coherence default.
