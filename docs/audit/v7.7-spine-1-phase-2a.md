R2 catches bugs INSIDE R1 fixes + adjacent surfaces. Examined the post-R1 surface:

- **C1 fix** (task_id template embedding): mechanical, single-file, test-pinned with regex assertion. No follow-up class to sweep.

Adjacent: are OTHER rituals in `src/rituals/` putting audit tools in `requiredTools`? `submit_report` is the ONLY audit tool, and morning.ts is the ONLY ritual that uses it in this phase. No adjacent sweep possible.
- **W1 fix** (cap counts `!= 'pass'`): SQL change with explicit test. No "follow-up cap-counting" surface to audit.
- **W2 fix** (sha256 optional): schema change. Adjacent: other citation variants — checked, none have a "producer can't compute the value" problem.
- **W3-W5, I1-I4 fixes**: all mechanical, all test-pinned.

R2 would have spent ~30-60 min finding ~0 new issues. **The decision to skip R2 is documented at the time of skipping** so a future reviewer can challenge it.
