## Architecture

The composed flow exercises the textbook hand-off:

```
synthetic returns matrix
      │
      ▼
HRP weights ──► equilibriumReturnsReverse ──► π (BL prior)
      │                                          │
      ▼                                          ▼
BM25 reflection retrieval     adversarial bull/bear/judge (stub infer)
                                                 │
                                                 ▼
                                      AssetSignals → BL combine
                                                 │
                                                 ▼
                                  posteriorMu + posteriorSigma
```

Three test blocks:

1. **Happy path with single view** — pins HRP→BL→BM25→adversarial→BL hand-off. Single positive AAPL view so the monotonicity assertion `posteriorMu[AAPL] > π[AAPL]` is a BL math guarantee, not a numeric coincidence.
2. **Two non-orthogonal views** — pins SHAPE/FINITENESS only when AAPL+0.04 and NVDA-0.01 views combine. No sign-of-update assertions (round-1 audit W1: cross-view interactions through Σ off-diagonals aren't a BL invariant).
3. **Fail-open path** — bull-side stub throws; asserts `judged === draft`, `failedOpen=true`, `failureReason` contains "bull" (round-2 W9), `calls === 2` exactly (round-2 W2 → bull throws + bear succeeds via `Promise.allSettled`, judge never invoked).

## Lessons

1. **"Aspirational contract" tests deserve explicit framing**. The composed test pins a flow with zero production callers (W4). If the framing is silent, future readers will mistake the test for catching real producer/consumer drift. The fix is documentation, not code: the docstring lists F7-landing actions ("update OR delete this file when F7 wires").

2. **Multi-view BL monotonicity is NOT a math guarantee**. A one-hot view on asset k pulls `posteriorMu[k]` toward `Q_k` (math guarantee). But cross-view interactions through Σ off-diagonals can flip the sign on the second-view asset. Round-1 W1 caught this — the test now asserts monotonicity only on a single-view sub-case.

3. **Trust-model precedents stick**. The composed test is reference material for future F7 wire-ups. Round-1 W5 caught that wiring `lessonsBlock` (LLM-summarized text) into `runAdversarialCritique.context` would establish a prompt-injection-vector precedent. The fix moves lessons to their own channel; the comment block now warns future authors against the unsafe pattern.

4. **`_seed` parameters that don't seed are footguns**. Round-1 W6 — the leading-underscore convention silenced eslint but the signature implied seeding intent that didn't exist. Drop unused parameters; don't underscore-prefix and call it done.

5. **Mid-bundle catches are part of the audit cycle**. The orthogonal-data error (cov = -2/3 instead of 0) was caught by `vitest` between round-2 fix application and commit. This is the failure mode round-2 audits are MEANT to catch — but in this case the type-checker + test runner caught it independently. Keep both lines of defense.
