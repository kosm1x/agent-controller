## Spec deviations (documented, intentional)

- **`summary_embedding` BLOB column, not `vec0` virtual table.** Codebase uses BLOB + in-memory cosine everywhere (`conversation_embeddings`, `skills.description_embedding`); sqlite-vec is "a future option when scale > 1000" (`src/skills/embedding.ts`). `general_events` is a ~30-50 row cohort. Build-once-use-twice → mirror skills.
- **`details_embedding` (spec §5 MIRIX multi-vector) omitted.** Descent runs through the explicit `general_event_episodic_links` table, not a second vector probe. Re-addable as an additive `ALTER TABLE` if a later spine wants 1-row/2-probe retrieval.
