## Post-deploy correction (W3 fold superseded)

The W3 fold above shipped `[ -f ./.env ] && { set -a && . ./.env && set +a; }`. Live deploy exposed that this is wrong: mc-ctl runs `set -euo pipefail`, and a `.env` value containing a `$N` token is a fatal unbound-variable under `set -u` when sourced. Replaced with an `env_value()` helper that greps a single key (`GEMINI_API_KEY`) out of `.env` without sourcing it — mirroring mc-ctl's existing `api_key()`. Shipped in the Spine 4 closure commit. Caught only because the W3 fold's removal of `2>/dev/null` made the failure loud — the fold's intent (surface `.env` problems) held; its mechanism was wrong. See `docs/planning/v7.7-spine-4-impl.md` §5.
